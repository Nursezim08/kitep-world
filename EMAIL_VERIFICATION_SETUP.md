# Настройка верификации Email

## Обзор

Система верификации email реализована для подтверждения email-адресов пользователей при регистрации. После регистрации пользователь получает 6-значный код на указанный email, который необходимо ввести для завершения регистрации.

## Компоненты

### 1. База данных

**Таблица `email_verifications`:**
- `id` - UUID идентификатор
- `userId` - ID пользователя
- `email` - Email адрес
- `code` - 6-значный код верификации
- `expiresAt` - Время истечения кода (10 минут)
- `verified` - Статус верификации
- `createdAt` - Время создания

**Поле в таблице `users`:**
- `emailVerified` - Boolean флаг верификации email

### 2. API Endpoints

#### POST `/api/auth/register`
Регистрация нового пользователя и отправка кода верификации.

**Request:**
```json
{
  "fullName": "Иван Иванов",
  "email": "user@example.com",
  "password": "SecurePass123",
  "phone": "+996555123456" // опционально
}
```

**Response:**
```json
{
  "message": "Регистрация успешна. Проверьте email для верификации.",
  "userId": "uuid",
  "email": "user@example.com",
  "requiresVerification": true
}
```

#### POST `/api/auth/verify-email`
Верификация email с помощью кода.

**Request:**
```json
{
  "userId": "uuid",
  "code": "123456"
}
```

**Response:**
```json
{
  "message": "Email успешно верифицирован",
  "user": {
    "id": "uuid",
    "fullName": "Иван Иванов",
    "email": "user@example.com",
    "emailVerified": true,
    ...
  }
}
```

#### POST `/api/auth/resend-code`
Повторная отправка кода верификации.

**Request:**
```json
{
  "userId": "uuid"
}
```

**Response:**
```json
{
  "message": "Код верификации отправлен повторно"
}
```

**Ограничения:**
- Не более 1 запроса в минуту на пользователя

### 3. Страницы

#### `/register`
Страница регистрации. После успешной регистрации перенаправляет на `/verify-email`.

#### `/verify-email?userId=xxx&email=xxx`
Страница ввода 6-значного кода верификации.

**Функции:**
- Автоматический переход между полями ввода
- Поддержка вставки кода (Ctrl+V)
- Автоматическая отправка при заполнении всех полей
- Кнопка повторной отправки кода с таймером (60 секунд)
- Валидация кода на сервере

### 4. Email сервис

**Файл:** `lib/email.ts`

**Функции:**
- `generateVerificationCode()` - Генерация 6-значного кода
- `sendVerificationEmail(email, code)` - Отправка email с кодом

## Настройка Email сервиса

### Текущая реализация

Система использует **Gmail SMTP** через nodemailer для отправки email.

### Быстрая настройка Gmail SMTP

1. **Включите двухфакторную аутентификацию** в Google аккаунте
2. **Создайте App Password**: https://myaccount.google.com/apppasswords
3. **Обновите `.env` файл**:
   ```env
   SMTP_USER="your-email@gmail.com"
   SMTP_PASS="your-app-password"
   ```
4. **Перезапустите сервер**: `npm run dev`

📖 **Подробная инструкция:** См. файл `GMAIL_SMTP_SETUP.md`

### Режим разработки (Fallback)

Если SMTP не настроен, коды верификации выводятся в консоль сервера:

```
==================================================
📧 EMAIL VERIFICATION CODE (Development Mode)
==================================================
To: user@example.com
Code: 123456
==================================================
```

## Безопасность

1. **Срок действия кода:** 10 минут
2. **Одноразовое использование:** Код помечается как `verified` после использования
3. **Rate limiting:** Не более 1 запроса на повторную отправку в минуту
4. **Валидация на сервере:** Все проверки выполняются на backend

## Поток пользователя

1. Пользователь заполняет форму регистрации
2. Система создает пользователя с `emailVerified: false`
3. Генерируется 6-значный код и отправляется на email
4. Пользователь перенаправляется на `/verify-email`
5. Пользователь вводит код из email
6. При правильном коде:
   - `emailVerified` устанавливается в `true`
   - Создается сессия (JWT token)
   - Пользователь перенаправляется в `/profile`

## Тестирование

### Development
Коды выводятся в консоль сервера. Проверьте терминал после регистрации.

### Production
Убедитесь, что:
1. Email сервис настроен и работает
2. Домен отправителя верифицирован
3. Email не попадают в спам
4. Шаблон письма корректно отображается
