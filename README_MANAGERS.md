## 🎯 Обзор

Система управления менеджерами позволяет администраторам:
- Просматривать список всех менеджеров
- Добавлять новых менеджеров
- Редактировать данные менеджеров
- Удалять менеджеров
- Назначать менеджеров на филиалы
- Управлять статусами менеджеров
- Искать менеджеров по различным критериям

## 🚀 Быстрый старт

### 1. Запустить проект

```bash
npm run dev
```

### 2. Войти в админ-панель

```
http://localhost:3000/admin/login
```

Нажать на пункт "Менеджеры" в боковой панели

- Добавить первого менеджера
- Назначить его на филиал
- Протестировать все функции

## 🔌 API

### Endpoints

#### Получить всех менеджеров
```http
GET /api/admin/managers
Authorization: admin_session cookie

Response: 200 OK
{
  "managers": [
    {
      "id": "uuid",
      "fullName": "Имя",
      "email": "email@example.com",
      "phone": "+996...",
      "status": "active",
      "branchUsers": [...]
    }
  ]
}
```

#### Создать менеджера
```http
POST /api/admin/managers
Authorization: admin_session cookie
Content-Type: application/json

{
  "fullName": "Имя",
  "email": "email@example.com",
  "phone": "+996...",
  "password": "password",
  "branchId": "uuid"
}

Response: 200 OK
{
  "message": "Manager created successfully",
  "manager": {...}
}
```

#### Обновить менеджера
```http
PATCH /api/admin/managers/{id}
Authorization: admin_session cookie
Content-Type: application/json

{
  "fullName": "Новое имя",
  "status": "inactive",
  ...
}

Response: 200 OK
{
  "message": "Manager updated successfully",
  "manager": {...}
}
```

#### Удалить менеджера
```http
DELETE /api/admin/managers/{id}
Authorization: admin_session cookie

Response: 200 OK
{
  "message": "Manager deleted successfully"
}
```

#### Получить филиалы
```http
GET /api/admin/branches
Authorization: admin_session cookie

Response: 200 OK
{
  "branches": [
    {
      "id": "uuid",
      "name": "Название",
      "city": "Город",
      "code": "BR001"
    }
  ]
}
```

## 🔒 Безопасность

### Реализованные меры

✅ **Аутентификация**
- Cookie-based сессии
- Проверка на каждом запросе
- Автоматический logout при истечении

✅ **Авторизация**
- Проверка роли администратора
- Доступ только для админов
- Защита всех endpoints

✅ **Пароли**
- Хеширование bcrypt
- Salt rounds: 10
- Никогда не хранятся в открытом виде

✅ **Валидация**
- Проверка на клиенте
- Проверка на сервере
- Проверка типов TypeScript

✅ **База данных**
- Защита от SQL-инъекций (Prisma)
- Уникальные ограничения
- Foreign key constraints
- Каскадное удаление

## 🌍 Локализация

### Поддерживаемые языки

- 🇷🇺 **Русский** (ru) - полностью
- 🇰🇬 **Кыргызский** (kg) - полностью

### Добавление нового языка

1. Создать файл `app/i18n/locales/{lang}/admin.json`
2. Скопировать структуру из `ru/admin.json`
3. Перевести все тексты
4. Добавить язык в `app/i18n/settings.ts`
