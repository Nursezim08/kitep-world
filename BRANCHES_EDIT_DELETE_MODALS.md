# Модальные окна редактирования и удаления филиалов

## ✅ Реализовано

Добавлены полноценные модальные окна для редактирования и удаления филиалов, идентичные по стилю и функциональности модальным окнам менеджеров.

### 📝 Модальное окно редактирования

#### Функциональность
- Открывается при клике на иконку карандаша (Edit)
- Предзаполнено текущими данными филиала
- Все поля редактируемые

#### Поля формы
1. **Название филиала** (обязательное)
2. **Код филиала** (обязательное, уникальное)
3. **Город** (обязательное)
4. **Район** (обязательное)
5. **Адрес** (обязательное)
6. **Телефон** (обязательное)
7. **Email** (обязательное)
8. **Статус** (CustomSelect):
   - Активен
   - Неактивен
   - Закрыт

#### Дизайн
- Компактная форма (как у менеджеров)
- Сетка 2 колонки для парных полей
- Кастомный Select для статуса
- Кнопки: "Отмена" и "Сохранить"
- Размер: `max-w-2xl` (шире чем у менеджеров из-за большего количества полей)

#### Валидация
- Все обязательные поля проверяются
- Уникальность кода проверяется на сервере
- Email валидируется браузером
- Телефон валидируется браузером

### 🗑️ Модальное окно удаления

#### Функциональность
- Открывается при клике на иконку корзины (Trash2)
- Показывает полную информацию о филиале
- Предупреждает о последствиях
- Показывает количество связанных данных

#### Отображаемая информация
1. **Карточка филиала**:
   - Аватар с первой буквой
   - Название филиала
   - Код филиала
   - Город и район

2. **Предупреждение**:
   - Красный блок с восклицательным знаком
   - Текст о невозможности отмены
   - Информация о связанных данных

3. **Счетчики связанных данных**:
   - ⚠️ Количество назначенных менеджеров
   - ⚠️ Количество заказов

#### Защита от удаления
На уровне API (не UI):
- Нельзя удалить филиал с назначенными менеджерами
- Нельзя удалить филиал с существующими заказами
- Возвращается ошибка с понятным сообщением

#### Дизайн
- Красная тема для опасного действия
- Иконка корзины в заголовке
- Информационные блоки с предупреждениями
- Кнопки: "Отмена" (серая) и "Удалить" (красная)

```typescript
const openEditModal = (branch: Branch) => {
  setSelectedBranch(branch);
  setEditFormData({
    name: branch.name,
    code: branch.code,
    city: branch.city,
    district: branch.district,
    address: branch.address,
    phone: branch.phone,
    email: branch.email,
    status: branch.status,

```

**Открытие модального окна удаления:**
```typescript
const openDeleteModal = (branch: Branch) => {
  setBranchToDelete(branch);
  setShowDeleteModal(true);
};
```

**Обработка редактирования:**
```typescript
const handleEditBranch = async (e: React.FormEvent) => {
  e.preventDefault();
  if (!selectedBranch) return;

  const response = await fetch(`/api/admin/branches/${selectedBranch.id}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(editFormData),
  });

  if (response.ok) {
    setShowEditModal(false);
    setSelectedBranch(null);
    fetchBranches(); // Обновляем список
  }
};
```

**Обработка удаления:**
```typescript
const handleDeleteBranch = async (branchId: string) => {
  const response = await fetch(`/api/admin/branches/${branchId}`, {
    method: 'DELETE',
  });

  if (response.ok) {
    setShowDeleteModal(false);
    setBranchToDelete(null);
    fetchBranches(); // Обновляем список
  } else {
    const data = await response.json();
    alert(data.error); // Показываем ошибку
  }
};
```

### 🧪 Тестирование

#### Редактирование
1. Открыть список филиалов
2. Кликнуть на иконку карандаша у любого филиала
3. Изменить данные (например, название)
4. Нажать "Сохранить"
5. Проверить, что изменения отобразились в списке

#### Удаление (успешное)
1. Открыть список филиалов
2. Кликнуть на иконку корзины у филиала без менеджеров и заказов
3. Прочитать предупреждение
4. Нажать "Удалить"
5. Проверить, что филиал исчез из списка

#### Удаление (с ошибкой)
1. Назначить менеджера на филиал
2. Попытаться удалить этот филиал
3. Увидеть предупреждение о количестве менеджеров
4. Нажать "Удалить"
5. Получить ошибку от API
6. Филиал остается в списке
