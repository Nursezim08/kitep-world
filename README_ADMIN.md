# 🔐 Админ-панель Kitep World

## Что реализовано

✅ **Двухфакторная аутентификация через Telegram**
- Админ вводит email и пароль
- Система отправляет 6-значный код в Telegram
- Админ вводит код и получает доступ

✅ **Защита маршрутов**
- Админы не могут заходить на пользовательские страницы
- Пользователи не могут заходить в админку
- Неавторизованные перенаправляются на страницу входа

✅ **Безопасность**
- JWT токены с истечением через 7 дней
- Временные коды (5 минут)
- Одноразовые коды
- Проверка роли и статуса аккаунта

✅ **Переводы**
- Русский язык
- Кыргызский язык

## Быстрая настройка

### 1️⃣ Создайте Telegram бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям
4. Сохраните **Bot Token** (например: `1234567890:ABCdefGHI...`)

### 2️⃣ Получите ваш Telegram User ID

1. Откройте [@userinfobot](https://t.me/userinfobot) в Telegram
2. Отправьте любое сообщение
3. Сохраните ваш **User ID** (например: `123456789`)

### 3️⃣ Сгенерируйте хеш пароля

```bash
node scripts/generate-password-hash.js admin123
```

Скопируйте полученный хеш.

### 4️⃣ Настройте базу данных

Выполните SQL запросы (замените значения на свои):

```sql
-- 1. Добавьте настройки Telegram
INSERT INTO settings (id, key, value) 
VALUES 
  (gen_random_uuid(), 'ADMIN_TELEGRAM_BOT_TOKEN', 'ВАШ_BOT_TOKEN'),
  (gen_random_uuid(), 'ADMIN_TELEGRAM_USER_ID', 'ВАШ_TELEGRAM_USER_ID')
ON CONFLICT (key) DO UPDATE SET value = EXCLUDED.value;

-- 2. Создайте админа (замените значения)
INSERT INTO users (
  id, full_name, email, email_verified, role, 
  password_hash, status, created_at, updated_at
)
VALUES (
  gen_random_uuid(),
  'Администратор',
  'admin@kitep.kg',
  true,
  'admin',
  'ВАШ_ХЕШ_ПАРОЛЯ',
  'active',
  NOW(),
  NOW()
);
```

**⚠️ Важно:** Telegram ID хранится только в таблице `settings` (ключ `ADMIN_TELEGRAM_USER_ID`), а не в таблице `users`!

### 5️⃣ Запустите бота

1. Найдите вашего бота в Telegram (по имени из шага 1)
2. Нажмите кнопку **Start**

### 6️⃣ Войдите в админку

1. Откройте `http://localhost:3000/admin/login`
2. Введите:
   - Email: `admin@kitep.kg`
   - Пароль: `admin123`
3. Проверьте Telegram - должен прийти код
4. Введите код на странице
5. Готово! 🎉

## Структура маршрутов

### Админские страницы
- `/admin/login` - Вход в админ-панель
- `/admin/verify` - Ввод кода из Telegram
- `/admin/dashboard` - Главная страница админки

### API
- `POST /api/admin/login` - Проверка email/пароля
- `POST /api/admin/verify-code` - Проверка кода
- `POST /api/admin/resend-code` - Повторная отправка кода

## Решение проблем

### ❌ Код не приходит в Telegram

**Решение:**
1. Убедитесь, что бот запущен (нажмите Start)
2. Проверьте `ADMIN_TELEGRAM_BOT_TOKEN` в таблице `settings`
3. Проверьте, что `telegram_id` пользователя совпадает с `ADMIN_TELEGRAM_USER_ID`
4. Проверьте логи сервера

### ❌ Ошибка "Telegram не настроен"

**Решение:**
1. Проверьте наличие записей в таблице `settings`:
   - `ADMIN_TELEGRAM_BOT_TOKEN`
   - `ADMIN_TELEGRAM_USER_ID`
2. Telegram ID хранится только в `settings`, а не в таблице `users`

### ❌ Ошибка "Неверный код"

**Решение:**
1. Проверьте, что код не истек (5 минут)
2. Убедитесь, что вводите последний полученный код
3. Попробуйте запросить код повторно

### ❌ Админ попадает на обычные страницы

**Решение:**
Проверьте, что файл `proxy.ts` находится в корне проекта и содержит правильную логику защиты маршрутов.
