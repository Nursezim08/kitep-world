# Настройка Gmail для работы с Timeweb

## Проблема
Gmail SMTP может блокировать отправку писем с серверов хостинга Timeweb из-за мер безопасности Google.

## Что изменено в коде (автоматически)

### 1. Переключение на порт 587 (STARTTLS)
```typescript
port: 587,        // Вместо 465 (SSL)
secure: false,    // false для STARTTLS
requireTLS: true, // Обязательное использование TLS
```

### 2. Увеличены таймауты
```typescript
connectionTimeout: 15000, // 15 секунд (было 5)
greetingTimeout: 15000,   // 15 секунд (было 5)
socketTimeout: 15000,     // 15 секунд (было 5)
```

### 3. Добавлены настройки совместимости
```typescript
tls: {
  rejectUnauthorized: false, // Игнорировать проблемы SSL
  ciphers: 'SSLv3',          // Поддержка старых протоколов
},
logger: true,  // Логирование соединения
debug: true,   // Детальная отладка
```

## Что нужно проверить в аккаунте Gmail

### Шаг 1: Проверка "Ненадежные приложения"
1. Откройте https://myaccount.google.com/security
2. Найдите раздел **"Небезопасный доступ к приложениям"** или **"Less secure app access"**
3. **Включите** доступ для ненадежных приложений

❗ **ВАЖНО:** Эта настройка может быть недоступна, если у вас включена двухфакторная аутентификация.

### Шаг 2: Проверка двухфакторной аутентификации
Если у вас включена 2FA (двухфакторная аутентификация):

1. Откройте https://myaccount.google.com/apppasswords
2. Создайте **новый пароль приложения**:
   - Выберите "Почта" в качестве приложения
   - Выберите "Другое устройство" в качестве устройства
   - Назовите его "Nur-Kitep Timeweb"
3. Google сгенерирует 16-значный пароль (например: `abcd efgh ijkl mnop`)
4. **ЗАМЕНИТЕ** `SMTP_PASS` в `.env.production` на этот новый пароль:
   ```
   SMTP_PASS="abcd efgh ijkl mnop"
   ```
   (Можно с пробелами или без: `abcdefghijklmnop`)

### Шаг 3: Проверка блокировки входа
1. Откройте https://myaccount.google.com/notifications
2. Проверьте, нет ли уведомлений о заблокированных попытках входа
3. Если есть - разрешите доступ

### Шаг 4: "Разблокировать капчу"
1. Откройте https://accounts.google.com/DisplayUnlockCaptcha
2. Нажмите "Продолжить"
3. Это временно разрешит доступ

## Если ничего не помогло

### Вариант 1: Использовать другой Gmail аккаунт
Попробуйте создать **новый Gmail аккаунт** специально для отправки:
1. Без двухфакторной аутентификации
2. С включенным "Доступ для ненадежных приложений"
3. Обновите `SMTP_USER` и `SMTP_PASS` в `.env.production`

### Вариант 2: Использовать SMTP хостинга Timeweb
Timeweb может предоставлять свой SMTP сервер:
1. Проверьте в панели Timeweb раздел "Email" или "SMTP"
2. Если есть - используйте его:
   ```
   SMTP_HOST="mail.twc.ru"  # или другой хост от Timeweb
   SMTP_PORT="587"
   SMTP_USER="your-email@twc.ru"
   SMTP_PASS="your-password"
   ```

### Вариант 3: Использовать Mail.ru SMTP
Mail.ru обычно лучше работает с русскими хостингами:
1. Создайте почту на Mail.ru (например: `nurkitep@mail.ru`)
2. Обновите `.env.production`:
   ```
   SMTP_HOST="smtp.mail.ru"
   SMTP_PORT="465"
   SMTP_USER="nurkitep@mail.ru"
   SMTP_PASS="ваш-пароль"
   ```
3. Код нужно будет немного изменить (я могу помочь)

## Детальные логи для диагностики

После деплоя проверьте логи Timeweb. Теперь там будет детальная информация:
- `logger: true` - логирование всех SMTP операций
- `debug: true` - детальная информация о соединении

Вы увидите точную причину, почему Gmail отклоняет соединение.
