# Интеграция S3 для загрузки изображений категорий

## Обзор

Реализована полная интеграция с S3-совместимым хранилищем для загрузки изображений категорий. Изображения больше не хранятся в базе данных в формате base64, а загружаются в S3 bucket.

## Архитектура

### Утилита S3 (`lib/s3.ts`)

**Функции:**

1. **`uploadImageToS3(base64Image, folder)`**
   - Принимает изображение в формате base64
   - Извлекает тип изображения (png, jpg, etc.)
   - Генерирует уникальное имя файла через UUID
   - Загружает в S3 с публичным доступом
   - Возвращает публичный URL изображения

2. **`deleteImageFromS3(imageUrl)`**
   - Принимает полный URL изображения
   - Извлекает ключ файла из URL
   - Удаляет файл из S3

3. **`isS3Image(url)`**
   - Проверяет, является ли URL изображением из S3
   - Используется для определения, нужно ли удалять старое изображение

4. **`isBase64Image(str)`**
   - Проверяет, является ли строка base64 изображением
   - Используется для определения, нужно ли загружать изображение в S3

- ACL: `public-read` (файлы публично доступны)

## Изменения в API

### POST /api/admin/categories

**Было:**
```typescript
image: image || null  // Сохранение base64 в БД
```

**Стало:**
```typescript
let imageUrl = image;
if (image && isBase64Image(image)) {
  imageUrl = await uploadImageToS3(image, 'categories');
}
// Сохранение URL в БД
```

### PATCH /api/admin/categories/[id]

**Логика обновления:**
1. Проверяем, является ли новое изображение base64
2. Если да - загружаем в S3
3. Удаляем старое изображение из S3 (если оно там было)
4. Сохраняем новый URL в БД

**Код:**
```typescript
if (isBase64Image(image)) {
  const newImageUrl = await uploadImageToS3(image, 'categories');
  
  if (existingCategory.image && isS3Image(existingCategory.image)) {
    await deleteImageFromS3(existingCategory.image);
  }
  
  updateData.image = newImageUrl;
}
```

### DELETE /api/admin/categories/[id]

**Дополнительная логика:**
1. Получаем категорию перед удалением
2. Помечаем категорию как удалённую (мягкое удаление)
3. Удаляем изображение из S3 (если оно там было)

**Код:**
```typescript
const category = await prisma.category.findUnique({
  where: { id },
  select: { image: true },
});

await prisma.category.update({
  where: { id },
  data: { status: 'deleted' },
});

if (category?.image && isS3Image(category.image)) {
  await deleteImageFromS3(category.image);
}
```

## Преимущества

✅ **Производительность:**
- База данных не раздувается от base64 изображений
- Быстрая загрузка изображений через CDN
- Меньше нагрузки на сервер приложения

✅ **Масштабируемость:**
- Неограниченное хранилище для изображений
- Легко добавить другие типы файлов (товары, баннеры)
- Возможность использовать CDN для ускорения

✅ **Управление:**
- Централизованное хранилище всех изображений
- Автоматическое удаление при удалении категории
- Уникальные имена файлов (UUID)

✅ **Безопасность:**
- Публичный доступ только к изображениям
- Приватные ключи доступа в .env
- Валидация формата и размера на клиенте

## Обработка ошибок

**Загрузка изображения:**
- Если загрузка в S3 не удалась, возвращается ошибка 500
- Транзакция не создаётся, категория не сохраняется

**Удаление старого изображения:**
- Если удаление не удалось, ошибка логируется
- Операция продолжается (новое изображение уже загружено)
- Старое изображение остаётся в S3 (можно очистить вручную)

**Удаление категории:**
- Если удаление изображения не удалось, ошибка логируется
- Категория помечается как удалённая
- Изображение остаётся в S3 (можно очистить вручную)

## Миграция существующих данных

Если в БД уже есть категории с base64 изображениями:

1. **Автоматическая миграция при редактировании:**
   - При первом редактировании категории
   - Изображение автоматически загрузится в S3
   - URL сохранится в БД

2. **Ручная миграция (скрипт):**
   ```typescript
   // Получить все категории с base64 изображениями
   const categories = await prisma.category.findMany({
     where: {
       image: { startsWith: 'data:image/' }
     }
   });
   
   // Загрузить каждое изображение в S3
   for (const category of categories) {
     const imageUrl = await uploadImageToS3(category.image, 'categories');
     await prisma.category.update({
       where: { id: category.id },
       data: { image: imageUrl }
     });
   }
   ```

## Тестирование

**Проверка загрузки:**
1. Создать новую категорию с изображением
2. Проверить, что в БД сохранён URL (не base64)
3. Проверить, что изображение доступно по URL
4. Проверить, что файл есть в S3 bucket

**Проверка обновления:**
1. Изменить изображение категории
2. Проверить, что старое изображение удалено из S3
3. Проверить, что новое изображение загружено
4. Проверить, что URL обновлён в БД

**Проверка удаления:**
1. Удалить категорию
2. Проверить, что статус = 'deleted'
3. Проверить, что изображение удалено из S3
