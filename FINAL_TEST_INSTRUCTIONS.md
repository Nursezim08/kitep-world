## 🧪 Пошаговое тестирование

### Шаг 1: Подготовка

1. **Откройте два окна браузера:**
   - Окно 1: Консоль разработчика (F12)
   - Окно 2: Терминал с сервером

2. **Очистите консоль браузера:**
   - F12 → Console → Иконка 🚫 (Clear console)

3. **Проверьте терминал сервера:**
   - Убедитесь, что сервер запущен
   - Нет ошибок при запуске

### Шаг 2: Откройте страницу заказов

1. **Перейдите на:** `http://localhost:3000/manager/orders`
2. **Войдите как менеджер** (если не авторизованы)
3. **Найдите заказ** со статусом "Оплачен" (синий бейдж)

### Шаг 3: Нажмите кнопку "Выдать"

1. **Кликните на кнопку "Выдать"**
2. **Проверьте консоль браузера** - должны появиться логи:

```javascript

[openConfirmModal] Order details loaded: {
  id: "...",
  orderNumber: "ORD-...",
  totalAmount: 180,
  user: {...},
  orderItems: [...]
}
```

3. **Проверьте терминал сервера** - должны появиться логи:

```

[MANAGER_ORDER_GET] Manager branch: xyz789...
[MANAGER_ORDER_GET] Order found, items count: 2
[MANAGER_ORDER_GET] Returning formatted order
```

## ❌ Возможные ошибки и решения

### Ошибка 1: "Cannot read properties of null"

**Причина:** Данные не загрузились или имеют неправильную структуру

**Решение:**
1. Проверьте консоль браузера
2. Проверьте Network tab → Response
3. Проверьте логи сервера

### Ошибка 2: Пустое модальное окно

**Причина:** `orderDetails` не установлен

**Решение:**
1. Проверьте консоль: `[openConfirmModal] Order details loaded`
2. Если нет - проверьте Network tab
3. Проверьте статус ответа (должен быть 200)

### Ошибка 3: "Branch not found"

**Причина:** Менеджер не привязан к филиалу

**Решение:**
```sql
-- Проверьте привязку
SELECT * FROM branch_users WHERE user_id = '{managerId}';

-- Если нет записи, добавьте
INSERT INTO branch_users (user_id, branch_id, created_at)
VALUES ('{managerId}', '{branchId}', NOW());
```

### Ошибка 4: "Order not found"

**Причина:** Заказ не принадлежит филиалу менеджера

**Решение:**
```sql
-- Проверьте заказ
SELECT id, order_number, branch_id, order_status 
FROM orders 
WHERE id = '{orderId}';

-- Проверьте филиал менеджера
SELECT branch_id FROM branch_users WHERE user_id = '{managerId}';

-- Они должны совпадать
```

### Ошибка 5: Изображения не загружаются

**Причина:** Неправильные URL или отсутствуют изображения

**Решение:**
```sql
-- Проверьте изображения товаров
SELECT p.id, p.sku, pi.image_url
FROM products p
LEFT JOIN product_images pi ON p.id = pi.product_id
WHERE p.id IN (
  SELECT product_id FROM order_items WHERE order_id = '{orderId}'
);
```


Если какой-то лог отсутствует - проблема на этом этапе.

### Проверка базы данных

```sql
-- Полная проверка заказа
SELECT 
  o.id,
  o.order_number,
  o.total,
  o.order_status,
  o.branch_id,
  u.full_name as customer_name,
  u.email as customer_email,
  b.name as branch_name,
  COUNT(oi.id) as items_count
FROM orders o
LEFT JOIN users u ON o.user_id = u.id
LEFT JOIN branches b ON o.branch_id = b.id
LEFT JOIN order_items oi ON o.id = oi.order_id
WHERE o.id = '{orderId}'
GROUP BY o.id, u.full_name, u.email, b.name;
```
