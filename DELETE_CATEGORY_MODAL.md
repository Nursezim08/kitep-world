# Модальное окно подтверждения удаления категории

## Обзор

Создано красивое модальное окно подтверждения удаления категории в стиле админской панели, заменяющее стандартный браузерный `confirm()`.

```
├─────────────────────────────────────┤
│                                     │

│ категорию                           │
```

## Интеграция

### Изменения в `CategoriesClient.tsx`

**Добавлены состояния:**
```typescript
const [isDeleteModalOpen, setIsDeleteModalOpen] = useState(false);
const [categoryToDelete, setCategoryToDelete] = useState<{ id: string; name: string } | null>(null);
```

**Новые функции:**
```typescript
// Открытие модального окна
const handleDeleteClick = (category: Category, e: React.MouseEvent) => {
  e.stopPropagation();
  const categoryName = getCategoryName(category, 'ru');
  setCategoryToDelete({ id: category.id, name: categoryName });
  setIsDeleteModalOpen(true);
};

// Подтверждение удаления
const handleDeleteConfirm = async () => {
  if (!categoryToDelete) return;
  
  // Удаление через API
  const response = await fetch(`/api/admin/categories/${categoryToDelete.id}`, {
    method: 'DELETE',
  });
  
  if (response.ok) {
    setIsDeleteModalOpen(false);
    setCategoryToDelete(null);
    fetchCategories();
  }
};

// Отмена удаления
const handleDeleteCancel = () => {
  setIsDeleteModalOpen(false);
  setCategoryToDelete(null);
};
```

**Рендеринг модального окна:**
```typescript
{isDeleteModalOpen && categoryToDelete && (
  <DeleteCategoryModal
    categoryName={categoryToDelete.name}
    onConfirm={handleDeleteConfirm}
    onCancel={handleDeleteCancel}
  />
)}
```

## Процесс удаления

### Шаг 1: Клик по кнопке удаления
```
Пользователь кликает на иконку корзины
    ↓
handleDeleteClick(category, e)
    ↓
Извлекается название категории (ru)
    ↓
Сохраняется в categoryToDelete
    ↓
Открывается модальное окно
```

### Шаг 2: Отображение модального окна
```
Показывается:
- Название категории
- Предупреждение о последствиях
- Кнопки "Отмена" и "Удалить"
```

### Шаг 3: Действия пользователя

**Вариант A: Отмена**
```
Клик на "Отмена" или backdrop
    ↓
handleDeleteCancel()
    ↓
Закрытие модального окна
    ↓
Очистка categoryToDelete
```

**Вариант B: Подтверждение**
```
Клик на "Удалить"
    ↓
handleDeleteConfirm()
    ↓
DELETE /api/admin/categories/[id]
    ↓
Удаление категории из БД (status = 'deleted')
    ↓
Удаление изображения из S3
    ↓
Закрытие модального окна
    ↓
Обновление списка категорий
```

## Сравнение

### Было (стандартный confirm)
```javascript
if (!confirm('Вы уверены, что хотите удалить эту категорию?')) {
  return;
}
```

**Проблемы:**
- ❌ Некрасивый браузерный диалог
- ❌ Не соответствует дизайну админки
- ❌ Нет информации о последствиях
- ❌ Не показывается название категории
- ❌ Разный вид в разных браузерах

### Стало (кастомное модальное окно)
```typescript
<DeleteCategoryModal
  categoryName={categoryToDelete.name}
  onConfirm={handleDeleteConfirm}
  onCancel={handleDeleteCancel}
/>
```

**Преимущества:**
- ✅ Красивый дизайн
- ✅ Соответствует стилю админки
- ✅ Чёткое предупреждение
- ✅ Показывается название категории
- ✅ Одинаковый вид везде


### Анимации
- Backdrop: плавное появление через `backdrop-blur-sm`
- Кнопки: `transition-colors` при hover
- Модальное окно: `shadow-2xl` для глубины

### Адаптивность
- `max-w-md` - максимальная ширина 448px
- `w-full` - занимает всю доступную ширину
- `p-4` - отступы на мобильных устройствах
