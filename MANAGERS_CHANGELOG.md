# Changelog: Управление менеджерами

## Дата: 2 мая 2026

### ✨ Новые возможности

#### 1. Раздел "Менеджеры" в админ-панели
- Добавлен новый пункт меню "Менеджеры" в боковую панель
- Расположен между "Филиалы" и "Отчеты"
- Навигация работает через клик по пункту меню

#### 2. Страница управления менеджерами (`/admin/managers`)
- Таблица со списком всех менеджеров
- Отображение информации:
  - Аватар и полное имя
  - Email
  - Телефон
  - Назначенный филиал
  - Статус (активен/неактивен/заблокирован)
- Поиск по имени, email и телефону
- Действия: редактирование и удаление

#### 3. Добавление менеджера
- Модальное окно с формой
- Поля:
  - Полное имя (обязательно)
  - Email (обязательно, уникальный)
  - Телефон (необязательно, уникальный)
  - Пароль (обязательно, с показом/скрытием)
  - Филиал (необязательно, выбор из списка)
- Валидация на клиенте и сервере
- Автоматическое хеширование пароля

#### 4. Редактирование менеджера
- Модальное окно с предзаполненными данными
- Возможность изменить:
  - Полное имя
  - Email
  - Телефон
  - Пароль (необязательно)
  - Статус
  - Филиал
- Проверка уникальности при изменении email/телефона

#### 5. Удаление менеджера
- Подтверждение перед удалением
- Каскадное удаление связей с филиалами

### 🔧 API Endpoints

#### Новые роуты:
```
GET    /api/admin/managers          - Получить список менеджеров
POST   /api/admin/managers          - Создать менеджера
PATCH  /api/admin/managers/[id]     - Обновить менеджера
DELETE /api/admin/managers/[id]     - Удалить менеджера
GET    /api/admin/branches          - Получить список филиалов
```

### 📝 Примечания

1. Менеджеры создаются с `emailVerified: true` (верификация не требуется)
2. Один менеджер может быть назначен только на один филиал
3. При удалении менеджера связи с филиалами удаляются автоматически
4. Пароль можно не указывать при редактировании (сохранится старый)
