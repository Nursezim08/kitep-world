# Исправление отображения иконок PWA (13.06.2026)

## Проблема
Логотип не появлялся при установке PWA на телефон.

## Причины

1. **`purpose: "maskable any"`** - требует специального формата иконок с safe zone
2. **Нет padding** - логотип растягивался на всю иконку без отступов
3. **Нет favicon** - отсутствовала базовая иконка сайта

## Решение

### 1. Изменён `manifest.json`
```json
"purpose": "any"  // было: "maskable any"
```

**Почему:** 
- `"maskable any"` требует специальный формат с safe zone (80% контента в центре)
- `"any"` работает на всех устройствах без специальных требований

### 2. Пересозданы иконки с padding
```javascript
// Скрипт генерации обновлён:
const logoSize = Math.round(size * 0.8);  // Логотип занимает 80%
const padding = Math.round((size - logoSize) / 2);  // 10% padding со всех сторон
```

**Результат:**
- ✅ Логотип занимает 80% от размера иконки
- ✅ 10% padding (отступ) со всех сторон
- ✅ Белый фон
- ✅ Центрирован

**Примеры:**
- 72×72: логотип 58×58, padding 7px
- 192×192: логотип 154×154, padding 19px
- 512×512: логотип 410×410, padding 51px

### 3. Добавлен favicon
```html
<link rel="icon" href="/favicon.png" type="image/png" />
```

Создан файл `public/favicon.png` (копия icon-192x192.png)

## Тестирование

### На телефоне (Android):

1. **Очистите кеш приложения:**
   ```
   Настройки → Приложения → Chrome → Хранилище → Очистить кеш
   ```

2. **Удалите старую версию PWA** (если установлена):
   - Удерживайте иконку на домашнем экране
   - Выберите "Удалить"

3. **Пересоберите приложение:**
   ```bash
   npm run build
   npm run start
   ```

4. **Откройте сайт в Chrome**

5. **Установите PWA заново:**
   - Через баннер внизу
   - Или меню → "Установить приложение"

6. **Проверьте иконку:**
   - На домашнем экране должен быть логотип с отступами
   - Иконка должна быть чёткой и центрированной

### На телефоне (iOS):

1. **Очистите кеш Safari:**
   ```
   Настройки → Safari → Очистить историю и данные
   ```

2. **Удалите старую версию** (если есть)

3. **Откройте сайт в Safari**

4. **Добавьте на домашний экран:**
   - Кнопка "Поделиться" 📤
   - "На экран Домой"
   - "Добавить"

5. **Проверьте иконку:**
   - Должен быть логотип с белым фоном
   - С небольшими отступами

### В DevTools:

1. **Chrome DevTools → Application → Manifest**
2. Проверьте секцию **Icons**:
   - Все иконки должны загружаться без ошибок
   - Purpose: `any` (не `maskable any`)
3. Нажмите на каждую иконку для предпросмотра

### Lighthouse:

1. **DevTools → Lighthouse**
2. **Progressive Web App** категория
3. Проверьте пункт **"Has a maskable icon"**:
   - Может быть warning (это нормально)
   - Главное - "Installable" должен быть ✅

## Различия: any vs maskable

### `"purpose": "any"`
✅ Работает на всех устройствах
✅ Простой формат
✅ Не требует специальной обработки
✅ Рекомендуется для большинства случаев

### `"purpose": "maskable any"`
⚠️ Требует safe zone (80% контента в центре)
⚠️ Специальный формат для Android Adaptive Icons
⚠️ Может обрезаться на некоторых устройствах
⚠️ Требует больше тестирования

## Troubleshooting

### Иконка всё ещё не отображается:

**Вариант 1: Очистите кеш**
```
Настройки → Приложения → Chrome → Хранилище → Очистить данные
```

**Вариант 2: Проверьте путь к иконкам**
```
http://your-domain.com/icons/icon-192x192.png
```
Должна открыться иконка в браузере

**Вариант 3: Проверьте manifest**
```
http://your-domain.com/manifest.json
```
Должен отдаваться JSON без ошибок

**Вариант 4: Пересоздайте иконки**
```bash
node scripts/generate-icons.mjs
npm run build
```

### Иконка размытая:

- Увеличьте исходный логотип (`public/logonur.png`)
- Рекомендуемый размер: минимум 512×512
- Пересоздайте иконки: `npm run generate:icons`

### Белый фон не нужен:

Измените в `scripts/generate-icons.mjs`:
```javascript
background: { r: 255, g: 255, b: 255, alpha: 1 }  // Белый непрозрачный
// на
background: { r: 0, g: 0, b: 0, alpha: 0 }  // Прозрачный
```
