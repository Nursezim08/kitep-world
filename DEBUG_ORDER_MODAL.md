# Отладка модального окна выдачи заказа

### 1. Обновлен API endpoint
**Файл:** `app/api/manager/orders/[id]/route.ts`

**Изменения:**
- ✅ Использованы правильные имена таблиц (`branch_users`, `orders`, `users`, `branches`)
- ✅ Использованы правильные имена полей (`user_id`, `branch_id`, `order_number`)
- ✅ Добавлено форматирование данных для фронтенда
- ✅ Исправлены связи между таблицами

### 2. Добавлено логирование


### 3. Улучшена обработка ошибок

- ✅ Кнопка "Попробовать снова"
- ✅ Детальные сообщения об ошибках

## Как проверить

### Шаг 1: Откройте консоль браузера
1. Нажмите F12
2. Перейдите на вкладку "Console"

### Шаг 2: Нажмите кнопку "Выдать"
1. Откройте страницу `/manager/orders`
2. Найдите заказ со статусом "Оплачен"
3. Нажмите кнопку "Выдать"

### Шаг 3: Проверьте логи
Вы должны увидеть:
```
[openConfirmModal] Opening modal for order: abc123...
[openConfirmModal] Response status: 200
[openConfirmModal] Order details loaded: {...}
```

## Возможные ошибки

### Ошибка 401: Unauthorized
**Причина:** Менеджер не авторизован
**Решение:** Войдите в систему заново

### Ошибка 404: Branch not found
**Причина:** Менеджер не привязан к филиалу
**Решение:** Проверьте таблицу `branch_users`

### Ошибка 404: Order not found
**Причина:** Заказ не найден или не принадлежит филиалу менеджера
**Решение:** Проверьте, что заказ существует и принадлежит филиалу

### Ошибка 500: Internal server error
**Причина:** Ошибка на сервере
**Решение:** Проверьте логи сервера в терминале

## Если проблема не решена

1. **Проверьте Network tab (F12 → Network):**
   - Найдите запрос к `/api/manager/orders/[id]`
   - Проверьте статус ответа
   - Проверьте тело ответа

2. **Проверьте логи сервера:**
   - Откройте терминал с запущенным сервером
   - Найдите строки с `[MANAGER_ORDER_GET]`
   - Проверьте наличие ошибок

3. **Проверьте базу данных:**
   - Убедитесь, что таблицы существуют
   - Проверьте наличие данных
   - Проверьте связи между таблицами

4. **Очистите кеш:**
   - Ctrl+Shift+R (жесткая перезагрузка)
   - Очистите cookies
   - Перезапустите сервер
