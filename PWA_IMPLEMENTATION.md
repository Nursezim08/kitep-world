# Реализация PWA (Progressive Web App) для Nur-Kitep

## Дата: 13.06.2026

**Реализовано полноценное PWA приложение с поддержкой оффлайн режима:**

## Созданные файлы

### 1. Манифест приложения
**`public/manifest.json`**
- Название: "Nur-Kitep - Канцелярский магазин"
- Короткое название: "Nur-Kitep"
- Режим отображения: standalone (как нативное приложение)
- Цвет темы: фиолетовый (#8b5cf6)
- Иконки: 8 размеров (72x72 до 512x512)
- Категории: shopping, business
- Язык: русский

### 2. Service Worker
**`public/sw.js`**
- Кеширование статических ресурсов
- Стратегия Network First для HTML
- Стратегия Cache First для статики
- Offline fallback страница
- Автоматическое обновление кеша
- Исключения: API, _next, chrome-extension

### 3. Регистрация Service Worker
**`app/register-sw.tsx`**
- Клиентский компонент для регистрации SW
- Работает только в production
- Проверка обновлений каждые 60 секунд
- Уведомление о новой версии
- Автоматическая перезагрузка при обновлении

### 4. Оффлайн страница
**`app/offline/page.tsx`**
- Красивая страница для оффлайн режима
- Иконка WifiOff
- Кнопка "Попробовать снова"
- Кнопка "На главную"
- Совет о добавлении на главный экран

### 5. Баннер установки PWA
**`app/components/InstallPWA.tsx`**
- Красивый баннер с призывом установить
- Появляется внизу экрана
- Кнопки "Установить" и "Отмена"
- Сохранение отказа в localStorage
- Фиолетовый градиентный дизайн

### 6. Скрипт генерации иконок
**`scripts/generate-icons.mjs`**
- Автоматическая генерация иконок из логотипа
- 8 размеров: 72, 96, 128, 144, 152, 192, 384, 512
- Использует sharp для обработки изображений
- Белый фон, aspect-ratio contain

## Важные замечания

1. **Service Worker работает только в production!**
   - В dev режиме SW не регистрируется
   - Для тестирования используйте `npm run build && npm run start`

2. **HTTPS обязателен для production!**
   - PWA требует HTTPS (или localhost)
   - Service Worker не работает на HTTP

3. **Иконки должны быть сгенерированы!**
   - Запустите `npm run generate:icons` перед деплоем
   - Проверьте наличие всех иконок в `public/icons/`

4. **Логотип источник:**
   - Используется `public/logonur.png`
   - Размер логотипа должен быть достаточно большим (рекомендуется 1024x1024)

5. **Кеширование:**
   - Статические ресурсы кешируются навсегда
   - HTML страницы обновляются при каждом запросе (Network First)
   - API запросы НЕ кешируются

## Troubleshooting

### Баннер не появляется:
```javascript
localStorage.removeItem('pwa-install-dismissed')
```

### Service Worker не обновляется:
- Измените CACHE_NAME в sw.js
- Или: DevTools → Application → Clear storage

### PWA не устанавливается:
- Проверьте manifest.json (без ошибок)
- Проверьте наличие всех иконок
- Убедитесь в активации Service Worker
- Используйте HTTPS
