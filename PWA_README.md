# 📱 PWA - Progressive Web App

## Nur-Kitep теперь устанавливается как приложение!

---

## 🎯 Что такое PWA?

**Progressive Web App** - это веб-приложение, которое:
- 📱 Устанавливается на устройство как обычное приложение
- ⚡ Работает быстро благодаря кешированию
- 🔌 Доступно оффлайн (без интернета)
- 🔔 Может отправлять уведомления (в будущем)
- 🚀 Обновляется автоматически

---

## ✨ Возможности

### ✅ Установка
- **Android:** Кнопка в браузере или баннер на сайте
- **iOS:** Кнопка "Поделиться" → "На экран Домой"
- **Desktop:** Иконка в адресной строке Chrome/Edge

### ✅ Оффлайн режим
- Главная страница доступна без интернета
- Автоматическое кеширование при первом посещении
- Специальная страница при отсутствии связи

### ✅ Как нативное приложение
- Работает в отдельном окне (без адресной строки)
- Иконка на главном экране устройства
- Splash screen при запуске
- Быстрый запуск из домашнего экрана

## 🚀 Установка для пользователей

### 📱 Android (Chrome/Samsung Internet):

1. Откройте сайт **nur-kitep.kg**
2. Внизу появится **баннер "Установите приложение"**
3. Нажмите **"Установить"**

**Или:**
1. Нажмите **⋮** (меню) в правом верхнем углу
2. Выберите **"Установить приложение"**
3. Нажмите **"Установить"** в диалоге

### 🍎 iOS (Safari):

1. Откройте сайт **nur-kitep.kg**
2. Нажмите кнопку **"Поделиться"** 📤 (внизу экрана)
3. Прокрутите вниз и выберите **"На экран Домой"**
4. Нажмите **"Добавить"**

### 💻 Desktop (Chrome/Edge):

1. Откройте сайт **nur-kitep.kg**
2. В адресной строке появится **иконка установки** ⊕
3. Нажмите на иконку
4. Нажмите **"Установить"**

**Или:**
1. Внизу страницы появится **баннер установки**
2. Нажмите **"Установить"**

## 👨‍💻 Для разработчиков

### Быстрый старт:

```bash
# 1. Установите зависимости
npm install

# 2. Сгенерируйте иконки (уже сделано)
npm run generate:icons

# 3. Соберите production версию
npm run build

# 4. Запустите сервер
npm run start

# 5. Откройте http://localhost:3000
```

## 🧪 Тестирование

### Chrome DevTools:

1. Откройте DevTools (F12)
2. Вкладка **Application**
3. Проверьте:
   - ✅ **Manifest** загружен
   - ✅ **Service Workers** активен
   - ✅ **Cache Storage** заполнен

### Lighthouse Audit:

1. DevTools → **Lighthouse**
2. Категория: **Progressive Web App**
3. Нажмите **Analyze page load**
4. Ожидаемая оценка: **100/100**

### Оффлайн тест:

1. DevTools → **Network**
2. ✅ Поставьте галочку **Offline**
3. Обновите страницу (F5)
4. Должна открыться страница `/offline`

## ⚙️ Технические детали

### Service Worker:

**Стратегии кеширования:**
- **Network First** для HTML страниц
- **Cache First** для статических ресурсов
- **Offline Fallback** при отсутствии сети

**Кешируемые ресурсы:**
```javascript
const STATIC_CACHE = [
  '/',              // Главная
  '/offline',       // Оффлайн страница
  '/manifest.json', // Манифест
  '/icons/*',       // Иконки
  '/logonur.png',   // Логотип
];
```

**Исключения:**
- API запросы (`/api/*`)
- Next.js файлы (`/_next/*`)
- Chrome extensions
- POST/PUT/DELETE запросы

### Автообновления:

- Проверка каждые **60 секунд**
- Уведомление о новой версии
- Автоматическая активация
- Перезагрузка страницы

## ❓ FAQ

### Почему Service Worker не работает в dev режиме?

Service Worker работает **только в production** по соображениям безопасности. Для тестирования используйте:

```bash
npm run build
npm run start
```

### Как обновить иконки?

1. Замените `public/logonur.png`
2. Запустите:
```bash
npm run generate:icons
```

### Как очистить кеш?

**В браузере:**
1. DevTools → Application
2. Clear storage → Clear site data

**В коде:**
Измените `CACHE_NAME` в `public/sw.js`:
```javascript
const CACHE_NAME = 'nur-kitep-v2'; // было v1
```

### Как добавить страницы в оффлайн кеш?

Отредактируйте `public/sw.js`:
```javascript
const STATIC_CACHE = [
  '/',
  '/offline',
  '/catalog',    // добавьте новые
  '/products',   // страницы здесь
];
```

### Нужен ли HTTPS?

**Да!** Service Worker требует HTTPS в production.
Только `localhost` может работать на HTTP.
