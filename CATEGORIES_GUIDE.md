# Руководство по работе с категориями

## Обзор

Система управления категориями позволяет создавать иерархическую структуру категорий и подкатегорий для товаров с поддержкой двух языков (русский и кыргызский).

## Доступ

Раздел "Категории" доступен в боковой панели админки:
- URL: `/admin/categories`
- Требуется роль: `admin`

## Основные функции

### 1. Просмотр категорий

- **Сетка карточек**: Категории отображаются в виде карточек с изображениями
- **Статистика**: На каждой карточке показано количество подкатегорий и товаров
- **Навигация**: Клик по категории с подкатегориями открывает их список
- **Breadcrumbs**: Хлебные крошки для навигации по уровням иерархии

### 2. Поиск категорий

- Поле поиска в верхней части страницы
- Поиск работает по названиям на обоих языках (русский и кыргызский)
- Результаты фильтруются в реальном времени

### 3. Добавление категории

**Шаги:**
1. Нажмите кнопку "Добавить категорию"
2. Загрузите изображение в формате PNG (до 5MB)
3. Введите название на русском языке
4. Введите название на кыргызском языке
5. Выберите статус (Активна/Неактивна)
6. Нажмите "Создать"

**Примечания:**
- Если вы находитесь внутри категории, новая категория будет создана как подкатегория
- Все поля обязательны для заполнения
- Изображение должно быть в формате PNG

### 4. Редактирование категории

**Шаги:**
1. Нажмите кнопку "Изменить" на карточке категории
2. Измените нужные поля
3. Нажмите "Сохранить"

**Что можно изменить:**
- Изображение
- Название на русском
- Название на кыргызском
- Статус

### 5. Удаление категории

**Шаги:**
1. Нажмите кнопку с иконкой корзины на карточке категории
2. Подтвердите удаление

**Ограничения:**
- Нельзя удалить категорию, у которой есть подкатегории
- Нельзя удалить категорию, к которой привязаны товары
- Удаление является "мягким" (статус меняется на `deleted`)

## Иерархия категорий

### Структура

```
Категория 1
├── Подкатегория 1.1
│   ├── Подкатегория 1.1.1
│   └── Подкатегория 1.1.2
└── Подкатегория 1.2

Категория 2
├── Подкатегория 2.1
└── Подкатегория 2.2
```

### Навигация

- **Корневой уровень**: Отображаются категории без родителя
- **Клик по категории**: Если есть подкатегории, открывается их список
- **Breadcrumbs**: Показывают текущий путь и позволяют вернуться на любой уровень

## API Endpoints

### GET /api/admin/categories
Получение списка категорий

**Query параметры:**
- `parentId` - ID родительской категории (или 'null' для корневых)
- `search` - Поисковый запрос

**Ответ:**
```json
[
  {
    "id": "uuid",
    "parentId": "uuid | null",
    "image": "base64 string",
    "status": "active | inactive | deleted",
    "translations": [
      {
        "id": "uuid",
        "locale": "ru",
        "name": "Название"
      },
      {
        "id": "uuid",
        "locale": "kg",
        "name": "Аталышы"
      }
    ],
    "_count": {
      "children": 5,
      "products": 120
    }
  }
]
```

### POST /api/admin/categories
Создание новой категории

**Body:**
```json
{
  "parentId": "uuid | null",
  "image": "base64 string",
  "status": "active | inactive",
  "translations": {
    "ru": { "name": "Название" },
    "kg": { "name": "Аталышы" }
  }
}
```

### GET /api/admin/categories/[id]
Получение категории по ID

### PATCH /api/admin/categories/[id]
Обновление категории

**Body:**
```json
{
  "image": "base64 string",
  "status": "active | inactive",
  "translations": {
    "ru": { "name": "Новое название" },
    "kg": { "name": "Жаңы аталышы" }
  }
}
```

### DELETE /api/admin/categories/[id]
Удаление категории (мягкое)

## База данных

### Таблица categories

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор |
| parentId | UUID (nullable) | ID родительской категории |
| image | Text (nullable) | Изображение в base64 |
| status | Enum | active, inactive, deleted |

### Таблица category_translations

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор |
| categoryId | UUID | ID категории |
| locale | Enum | ru, kg |
| name | String | Название категории |

## Валидация

### При создании/редактировании:
- ✅ Изображение обязательно
- ✅ Формат изображения: PNG
- ✅ Максимальный размер: 5MB
- ✅ Название на русском обязательно
- ✅ Название на кыргызском обязательно

### При удалении:
- ❌ Нельзя удалить категорию с подкатегориями
- ❌ Нельзя удалить категорию с товарами

## Примеры использования

### Создание основной категории "Книги"

1. Перейдите в `/admin/categories`
2. Нажмите "Добавить категорию"
3. Загрузите изображение книги (PNG)
4. Введите "Книги" (русский)
5. Введите "Китептер" (кыргызский)
6. Выберите статус "Активна"
7. Нажмите "Создать"

### Создание подкатегории "Художественная литература"

1. Откройте категорию "Книги" (кликните по карточке)
2. Нажмите "Добавить категорию"
3. Загрузите изображение
4. Введите "Художественная литература" (русский)
5. Введите "Көркөм адабият" (кыргызский)
6. Нажмите "Создать"

### Поиск категории

1. Введите название в поле поиска
2. Результаты отфильтруются автоматически
3. Поиск работает на любом уровне иерархии

## Советы

- 📸 Используйте квадратные изображения для лучшего отображения
- 🎨 Изображения должны быть в формате PNG для поддержки прозрачности
- 🌍 Всегда заполняйте названия на обоих языках
- 📊 Проверяйте статистику перед удалением категории
- 🔍 Используйте поиск для быстрого нахождения нужной категории

## Устранение проблем

### Ошибка "Допускаются только PNG изображения"
- Убедитесь, что файл имеет расширение .png
- Конвертируйте изображение в PNG формат

### Ошибка "Размер изображения не должен превышать 5MB"
- Сожмите изображение
- Используйте онлайн-сервисы для оптимизации PNG

### Ошибка "Невозможно удалить категорию с подкатегориями"
- Сначала удалите все подкатегории
- Или переместите подкатегории в другую категорию

### Ошибка "Невозможно удалить категорию с товарами"
- Переместите товары в другую категорию
- Или удалите товары из этой категории

## Дополнительная информация

- Изображения хранятся в base64 в базе данных
- Поддерживается неограниченная вложенность категорий
- Удаление является "мягким" (можно восстановить через БД)
- Статистика обновляется автоматически при изменении данных
