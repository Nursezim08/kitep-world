# Анимация переназначения менеджеров с отложенным сохранением

## Описание

Реализована плавная анимация перехода менеджеров между филиалами с отложенным сохранением изменений.

## Ключевая особенность

**Визуальное перемещение ≠ Реальное переназначение**

- При клике на стрелку менеджеры **визуально** перемещаются между столбиками
- Реальное переназначение в БД происходит только при нажатии кнопки **"Сохранить"**
- Можно переместить несколько групп менеджеров перед сохранением
- Индикатор "Сохранение..." на кнопке во время процесса

## Как работает

### 1. Визуальный процесс

```
Филиал А                  →                  Филиал Б
┌─────────────┐                           ┌─────────────┐
│ Менеджер 1  │ ──────────────────────→  │             │
│ Менеджер 2  │ (анимация 800ms)         │ Менеджер 1  │
│ Менеджер 3  │                           │ Менеджер 2  │
└─────────────┘                           └─────────────┘
                                          
                    ↓ Нажать "Сохранить"
                    
                  API запрос → БД обновлена
                  Индикатор "Сохранение..." ⟳
```

### 2. Технический процесс

1. **Выбор менеджеров** - пользователь отмечает чекбоксами
2. **Клик на стрелку** - запускается визуальное перемещение
3. **Анимация выезда** - карточки плавно уезжают вправо (800ms)
4. **Анимация въезда** - карточки появляются слева в целевом филиале (800ms)
5. **Обновление визуального состояния** - `visualBranches` обновляется
6. **Сохранение в pendingReassignments** - изменения накапливаются
7. **Клик "Сохранить"** - все изменения применяются через API
8. **Индикатор "Сохранение..."** - на кнопке во время процесса
9. **Закрытие модального окна** - данные обновляются на странице

## Состояния компонента

### pendingReassignments
```typescript
{
  fromBranchId: string;
  toBranchId: string;
  managerIds: string[];
}[]
```
- Массив незафиксированных изменений
- Накапливает все перемещения до сохранения
- Используется для batch-обновления при сохранении

### visualBranches
- Локальная копия данных филиалов для визуального отображения
- Обновляется сразу после анимации
- Не влияет на реальные данные до сохранения

### animatingManagerIds
- Массив ID менеджеров, которые в данный момент анимируются
- Используется для применения CSS классов анимации
- Блокирует взаимодействие с карточкой во время анимации

## Информационные блоки

### Незафиксированные изменения (синий)
```
ℹ️ Незафиксированные изменения
Переназначено менеджеров: 3. Нажмите "Сохранить", чтобы применить изменения.
```

## Процесс сохранения

```typescript
const handleSave = async () => {
  // 1. Проверка наличия изменений
  if (pendingReassignments.length === 0) {
    onClose();
    return;
  }

  // 2. Выполнение всех переназначений последовательно
  for (const reassignment of pendingReassignments) {
    await fetch('/api/admin/branches/reassign-managers', {
      method: 'POST',
      body: JSON.stringify(reassignment),
    });
  }

  // 3. Показ toast уведомления
  showToast('Успешно переназначено менеджеров: X', 'success');

  // 4. Закрытие модального окна через 1 секунду
  setTimeout(() => {
    onSuccess(); // Обновление данных на странице
  }, 1000);
};
```

## Кнопка "Сохранить"

### Состояния кнопки:

**Нет изменений:**
```
[  Закрыть  ]  (серая)
```

**Есть изменения:**
```
[✓ Сохранить изменения]  (зеленая)
```

**Процесс сохранения:**
```
[⟳ Сохранение...]  (серая, disabled)
```

## Преимущества

✅ **Визуальная обратная связь** - пользователь видит процесс перемещения
✅ **Отложенное сохранение** - можно переместить несколько групп перед сохранением
✅ **Возможность отмены** - можно закрыть модальное окно без сохранения
✅ **Batch-обновление** - все изменения применяются за один раз
✅ **Всплывающее уведомление** - не отвлекает от работы
✅ **Плавный UX** - нет резких изменений интерфейса
