# Система кодов заказов

## Описание

Реализована система кодов для безопасной выдачи заказов клиентам.

## Как работает код заказа

### Генерация кода

Код заказа = **последние 5 цифр** номера заказа (без дефисов)

**Примеры:**
- Номер заказа: `ORD-1779710728185-808`
- Убираем дефисы: `ORD1779710728185808`
- Убираем буквы: `1779710728185808`
- Берем последние 5 цифр: **`85808`**

### Процесс выдачи заказа (менеджер)

1. **Открытие модального окна:**
   - Менеджер нажимает кнопку "Выдать" на заказе со статусом `paid`
   - Открывается модальное окно с деталями заказа

2. **Информация в модальном окне:**
   - ✅ Данные клиента (имя, email, телефон)
   - ✅ Список всех товаров с изображениями
   - ✅ Количество и цена каждого товара
   - ✅ Общая сумма заказа
   - ✅ Поле ввода кода подтверждения

3. **Ввод кода:**
   - Менеджер просит клиента назвать код заказа
   - Клиент открывает приложение и показывает код
   - Менеджер вводит 5 цифр в поле
   - Кнопка "Выдать заказ" активна только при вводе 5 цифр

4. **Подтверждение:**
   - При правильном коде → заказ переводится в статус `completed`
   - При неправильном коде → показывается ошибка "Неверный код заказа"

### Просмотр кода (пользователь)

1. **Открытие деталей заказа:**
   - Пользователь переходит на страницу `/orders`
   - Кликает по карточке заказа
   - Открывается модальное окно с деталями

2. **Информация в модальном окне:**
   - ✅ **Код для получения заказа** (крупный, в фиолетовом блоке)
   - ✅ Статус заказа
   - ✅ Информация о филиале
   - ✅ Список всех товаров
   - ✅ Общая сумма

3. **Использование кода:**
   - Пользователь приходит в филиал
   - Называет код менеджеру
   - Получает свой заказ

## Технические детали

### Функция извлечения кода

```typescript
const getOrderCode = (orderNumber: string) => {
  // Убираем дефисы и все нецифровые символы
  const digits = orderNumber.replace(/-/g, '').replace(/\D/g, '');
  // Берем последние 5 цифр
  return digits.slice(-5);
};
```

### API Endpoint

**`GET /api/manager/orders/[id]`**

Возвращает детальную информацию о заказе:
- Данные клиента
- Список товаров с изображениями и переводами
- Информация о филиале
- Информация о платеже

**Безопасность:**
- Менеджер видит только заказы своего филиала
- Проверка через таблицу `branch_users`

## Преимущества системы

✅ **Безопасность:** Защита от выдачи заказа не тому клиенту
✅ **Простота:** Всего 5 цифр, легко запомнить
✅ **Удобство:** Код всегда доступен в приложении
✅ **Прозрачность:** Клиент видит все детали заказа
✅ **Контроль:** Менеджер проверяет код перед выдачей

## Статусы заказов

- `paid` (Оплачен) - заказ оплачен, ожидает выдачи
- `completed` (Завершен) - заказ выдан клиенту
- `cancelled` (Отменен) - заказ отменен

## Интерфейс

### Менеджер (светлая тема)
- Модальное окно с деталями заказа
- Поле ввода кода (5 цифр, центрированное)
- Кнопка "Выдать заказ" (зеленая)
- Валидация в реальном времени

### Пользователь (светлая тема)
- Модальное окно с деталями заказа
- Крупный код в фиолетовом блоке (text-4xl)
- Подсказка: "Назовите этот код менеджеру"
- Список товаров с изображениями
- Общая сумма заказа
