# Реализация раздела "Филиалы"

## ✅ Что реализовано

### 📁 Структура файлов

#### API Endpoints
- `app/api/admin/branches/route.ts` - GET (список), POST (создание)
- `app/api/admin/branches/[id]/route.ts` - GET (детали), PATCH (обновление), DELETE (удаление)

#### Страницы
- `app/admin/branches/page.tsx` - Серверный компонент с авторизацией
- `app/admin/branches/BranchesClient.tsx` - Клиентский компонент со всей логикой

### 🎯 Функциональность

#### 1. Список филиалов
- Таблица с информацией о филиалах
- Колонки: Филиал, Код, Город, Контакты, Менеджеры, Статус, Действия
- Клик по строке → переход на страницу деталей филиала
- Аватар с первой буквой названия
- Иконки для визуализации (код, город, телефон, email)

#### 2. Поиск
- Текстовый поиск по:
  - Названию филиала
  - Коду филиала
  - Городу
  - Телефону
- Работает в реальном времени

#### 3. Фильтры
- **По статусу**: Все / Активен / Неактивен / Закрыт
- **По городу**: Все города / Список уникальных городов
- Кнопка "Фильтры" с счетчиком активных фильтров
- Визуальные теги активных фильтров с возможностью удаления
- Кнопка "Сбросить" для очистки всех фильтров

#### 4. Сортировка
- **Поля**: Дата создания / Название / Код / Город
- **Порядок**: По возрастанию / По убыванию
- Клик по заголовкам таблицы для быстрой сортировки
- Иконки стрелок показывают направление

#### 5. Пагинация
- 50 филиалов на страницу
- Кнопки: Первая, Предыдущая, Номера страниц, Следующая, Последняя
- Умное отображение номеров страниц с многоточием
- Счетчик: "Показано X-Y из Z филиалов"
- Автоматический сброс на страницу 1 при изменении фильтров

#### 6. Добавление филиала
- Модальное окно с формой
- Обязательные поля:
  - Название филиала
  - Код филиала (уникальный)
  - Город
  - Район
  - Адрес
  - Телефон
  - Email
- Валидация на клиенте и сервере
- Проверка уникальности кода

#### 7. Редактирование филиала
- Модальное окно с предзаполненными данными
- Возможность изменить все поля
- Изменение статуса (Активен/Неактивен/Закрыт)
- Проверка уникальности кода при изменении

#### 8. Удаление филиала
- Модальное окно подтверждения
- Отображение информации о филиале
- Предупреждение о последствиях
- Защита от удаления:
  - Если есть назначенные менеджеры
  - Если есть существующие заказы

### 📊 База данных

#### Модель Branch
```prisma
model Branch {
  id          String        @id @default(uuid())
  name        String        // Название
  code        String        @unique // Уникальный код
  city        String        // Город
  district    String        // Район
  address     String        // Адрес
  phone       String        // Телефон
  email       String        // Email
  latitude    Decimal?      // Широта (опционально)
  longitude   Decimal?      // Долгота (опционально)
  openTime    DateTime?     // Время открытия (опционально)
  closeTime   DateTime?     // Время закрытия (опционально)
  status      BranchStatus  @default(active)
  createdAt   DateTime      @default(now())
  updatedAt   DateTime      @updatedAt
  
  // Relations
  branchUsers   BranchUser[]
  inventory     BranchInventory[]
  orders        Order[]
}
```

#### Статусы
- `active` - Активен
- `inactive` - Неактивен
- `closed` - Закрыт

### 🔄 API Endpoints

#### GET /api/admin/branches
- Получить список всех филиалов
- Включает: branchUsers, _count (branchUsers, orders)
- Сортировка по дате создания (desc)

#### POST /api/admin/branches
- Создать новый филиал
- Валидация обязательных полей
- Проверка уникальности кода
- Возвращает созданный филиал

#### GET /api/admin/branches/[id]
- Получить детали филиала
- Включает: branchUsers с данными пользователей, последние 10 заказов, счетчики

#### PATCH /api/admin/branches/[id]
- Обновить филиал
- Частичное обновление (только переданные поля)
- Проверка уникальности кода при изменении

#### DELETE /api/admin/branches/[id]
- Удалить филиал
- Проверка на наличие связанных менеджеров
- Проверка на наличие заказов
- Возвращает ошибку если есть связанные данные

## Использование

1. Перейти в `/admin/branches`
2. Просмотреть список филиалов
3. Использовать поиск и фильтры
4. Добавить новый филиал через кнопку "Добавить филиал"
5. Редактировать через иконку карандаша
6. Удалить через иконку корзины
7. Кликнуть на строку для перехода к деталям (когда будет реализовано)
