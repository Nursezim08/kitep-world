# 🌱 Seed данные для Nur-Kitep

## Быстрый старт

```bash
npm run db:seed
```

## 📦 Что включено

- **10 категорий** товаров с изображениями
- **100 товаров** (по 10 в каждой категории)
- **400 изображений** товаров

## 🎯 Особенности

✅ **Уникальные SKU** - каждый товар имеет уникальный артикул (КАН-0001, КНИ-0002)  
✅ **Реалистичные данные** - названия, описания, бренды  
✅ **Мультиязычность** - переводы на русском и кыргызском  
✅ **Изображения** - 4 изображения для каждого товара  
✅ **Готово к работе** - можно сразу тестировать функционал

## 🛠️ Настройка

### Изменение изображений

Откройте `prisma/seed.ts` и измените массив `productImages`:

```typescript
const productImages = [
  'https://your-s3-bucket.com/image1.jpg',
  'https://your-s3-bucket.com/image2.jpg',
  'https://your-s3-bucket.com/image3.jpg',
  'https://your-s3-bucket.com/image4.jpg',
];
```

### Добавление категорий

Добавьте новую категорию в массив `categories`:

```typescript
{
  nameRu: 'Новая категория',
  nameKg: 'Жаңы категория',
  image: 'https://example.com/image.png',
}
```

И создайте товары в `productTemplates`:

```typescript
'Новая категория': [
  { nameRu: 'Товар 1', nameKg: 'Товар 1', brand: 'Brand', price: 100 },
  // ... ещё 9 товаров
]
```

### Отключение очистки данных

Если не хотите удалять существующие данные, закомментируйте в `prisma/seed.ts`:

```typescript
// await prisma.productImage.deleteMany();
// await prisma.productTranslation.deleteMany();
// await prisma.product.deleteMany();
// await prisma.categoryTranslation.deleteMany();
// await prisma.category.deleteMany();
```

### Замена на реальные изображения

**Вариант 1: Через админку**
1. Откройте товар в админке
2. Нажмите "Редактировать"
3. Загрузите новые изображения
4. Сохраните

**Вариант 2: Через seed скрипт**
1. Загрузите изображения в S3
2. Обновите URL в `prisma/seed.ts`
3. Запустите `npm run db:seed`

## 🔧 Troubleshooting

### Ошибка: "Cannot find module 'tsx'"
```bash
npm install
```

### Ошибка: "Database connection failed"
Проверьте `DATABASE_URL` в `.env`

### Ошибка: "Foreign key constraint failed"
Убедитесь, что база данных пустая или отключите очистку данных

### Изображения не загружаются
Замените URL в `prisma/seed.ts` на доступные изображения

## 💡 Советы

1. **Запускайте seed после каждого сброса БД**
   ```bash
   npx prisma migrate reset
   npm run db:seed
   ```

2. **Используйте Prisma Studio для просмотра данных**
   ```bash
   npx prisma studio
   ```

3. **Создайте резервную копию перед seed**
   ```bash
   pg_dump your_database > backup.sql
   ```

4. **Добавьте остатки товаров после seed**
   - Через админку: Товары → Редактировать → Остатки
   - Или создайте отдельный seed для `branch_inventory`
