# Пагинация списка менеджеров

## Описание

Добавлена полноценная система пагинации для списка менеджеров с ограничением в 50 записей на страницу.

## Основные характеристики

### 📊 Параметры
- **Записей на страницу**: 50 менеджеров
- **Автоматический расчет**: Количество страниц вычисляется динамически
- **Умная навигация**: Показываются только релевантные номера страниц

### 🎯 Функциональность

#### Кнопки навигации
1. **⏮️ Первая страница** (ChevronsLeft)
   - Переход на страницу 1
   - Неактивна на первой странице

2. **◀️ Предыдущая страница** (ChevronLeft)
   - Переход на предыдущую страницу
   - Неактивна на первой странице

3. **Номера страниц**
   - Текущая страница выделена фиолетовым
   - Показываются: первая, последняя, текущая ±1
   - Многоточие (...) между пропущенными страницами

4. **▶️ Следующая страница** (ChevronRight)
   - Переход на следующую страницу
   - Неактивна на последней странице

5. **⏭️ Последняя страница** (ChevronsRight)
   - Переход на последнюю страницу
   - Неактивна на последней странице

### 📍 Расположение

Пагинация находится в **нижней части таблицы**:
```
┌─────────────────────────────────────────────────────┐
│  Таблица с менеджерами                              │
├─────────────────────────────────────────────────────┤
│ ⏮️ ◀️ [1] 2 3 ... 10 ▶️ ⏭️    Показано 1-50 из 500 │
└─────────────────────────────────────────────────────┘
```

**Левая часть**: Кнопки навигации и номера страниц  
**Правая часть**: Счетчик результатов

### 🎨 Дизайн

#### Кнопки навигации
- **Обычное состояние**: Серый цвет (`text-gray-400`)
- **Hover**: Белый текст + фон `#2a3347`
- **Неактивная**: Прозрачность 30%, курсор not-allowed
- **Размер иконок**: 18px

#### Номера страниц
- **Текущая страница**: 
  - Фон: `bg-violet-500`
  - Текст: белый
  - Жирный шрифт
- **Другие страницы**:
  - Текст: серый (`text-gray-400`)
  - Hover: белый текст + фон `#2a3347`
- **Размер**: минимум 36px ширина, padding `px-3 py-1.5`

#### Многоточие
- Цвет: `text-gray-500`
- Padding: `px-2`
- Не кликабельное

### 🔄 Автоматический сброс

При изменении любого из следующих параметров, пагинация автоматически возвращается на **страницу 1**:
- Поисковый запрос
- Фильтр по статусу
- Фильтр по филиалу
- Поле сортировки
- Порядок сортировки

Это реализовано через `useEffect`:
```typescript
useEffect(() => {
  setCurrentPage(1);
}, [searchQuery, statusFilter, branchFilter, sortBy, sortOrder]);
```

### 📈 Счетчик результатов

Формат: **"Показано X-Y из Z менеджеров"**

Примеры:
- Страница 1 из 3 (150 записей): "Показано 1-50 из 150 менеджеров"
- Страница 2 из 3 (150 записей): "Показано 51-100 из 150 менеджеров"
- Страница 3 из 3 (150 записей): "Показано 101-150 из 150 менеджеров"
- Единственная страница (20 записей): "Показано 1-20 из 20 менеджеров"

### 🎯 Состояния компонента

```typescript
const [currentPage, setCurrentPage] = useState(1);
const [itemsPerPage] = useState(50);

// Вычисления
const totalPages = Math.ceil(sortedManagers.length / itemsPerPage);
const startIndex = (currentPage - 1) * itemsPerPage;
const endIndex = startIndex + itemsPerPage;
const paginatedManagers = sortedManagers.slice(startIndex, endIndex);
```

### 🔧 Технические детали

#### Расчет страниц
```typescript
// Общее количество страниц
const totalPages = Math.ceil(sortedManagers.length / itemsPerPage);

// Индексы для slice
const startIndex = (currentPage - 1) * itemsPerPage;
const endIndex = startIndex + itemsPerPage;

// Данные для текущей страницы
const paginatedManagers = sortedManagers.slice(startIndex, endIndex);
```

#### Фильтрация номеров страниц
```typescript
Array.from({ length: totalPages }, (_, i) => i + 1)
  .filter(page => {
    return (
      page === 1 ||                              // Первая
      page === totalPages ||                     // Последняя
      (page >= currentPage - 1 && page <= currentPage + 1)  // Текущая ±1
    );
  })
```

### 📱 Адаптивность

#### Мобильные устройства
- Кнопки навигации остаются видимыми
- Номера страниц могут переноситься
- Счетчик результатов может переноситься под кнопки

#### Планшеты и десктоп
- Все элементы в одну строку
- Оптимальное использование пространства

### ✨ Преимущества

1. **Производительность**: Рендерится только 50 записей вместо всех
2. **UX**: Легкая навигация по большим спискам
3. **Визуальная ясность**: Понятно, где находишься и сколько всего страниц
4. **Умная навигация**: Быстрый переход в начало/конец списка
5. **Автоматический сброс**: При фильтрации всегда начинаешь с первой страницы

### 📊 Примеры использования

#### Малое количество (< 50)
- Пагинация не отображается
- Показываются все записи
- Счетчик: "Показано 1-20 из 20 менеджеров"

#### Среднее количество (50-200)
- 2-4 страницы
- Все номера страниц видны
- Легкая навигация

#### Большое количество (> 200)
- 5+ страниц
- Умное отображение с многоточием
- Быстрый переход в начало/конец
