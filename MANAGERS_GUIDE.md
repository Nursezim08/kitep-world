# Руководство по управлению менеджерами

## Обзор

Добавлена новая функциональность для управления менеджерами филиалов в админ-панели. Менеджеры могут быть назначены на конкретные филиалы и управлять их работой.

## Структура

### API Endpoints

#### 1. `/api/admin/managers` (GET)
Получение списка всех менеджеров с информацией о филиалах.

**Ответ:**
```json
{
  "managers": [
    {
      "id": "uuid",
      "fullName": "Имя менеджера",
      "email": "email@example.com",
      "phone": "+996XXXXXXXXX",
      "status": "active",
      "createdAt": "2024-01-01T00:00:00.000Z",
      "branchUsers": [
        {
          "branch": {
            "id": "uuid",
            "name": "Название филиала",
            "city": "Город"
          }
        }
      ]
    }
  ]
}
```

#### 2. `/api/admin/managers` (POST)
Создание нового менеджера.

**Тело запроса:**
```json
{
  "fullName": "Имя менеджера",
  "email": "email@example.com",
  "phone": "+996XXXXXXXXX",
  "password": "password123",
  "branchId": "uuid" // необязательно
}
```

**Особенности:**
- Email должен быть уникальным
- Телефон должен быть уникальным (если указан)
- Пароль хешируется перед сохранением
- Менеджеры создаются с `emailVerified: true`
- Статус по умолчанию: `active`
- Роль: `manager`

#### 3. `/api/admin/managers/[id]` (PATCH)
Обновление данных менеджера.

**Тело запроса:**
```json
{
  "fullName": "Новое имя",
  "email": "newemail@example.com",
  "phone": "+996XXXXXXXXX",
  "password": "newpassword", // необязательно
  "status": "active", // active | inactive | blocked
  "branchId": "uuid" // необязательно, null для удаления привязки
}
```

**Особенности:**
- Можно обновить любое поле
- Пароль обновляется только если указан
- При изменении филиала старая привязка удаляется
- Проверка уникальности email и телефона

#### 4. `/api/admin/managers/[id]` (DELETE)
Удаление менеджера.

**Особенности:**
- Каскадное удаление связей с филиалами
- Проверка существования менеджера

#### 5. `/api/admin/branches` (GET)
Получение списка активных филиалов для выбора при создании/редактировании менеджера.

**Ответ:**
```json
{
  "branches": [
    {
      "id": "uuid",
      "name": "Название филиала",
      "city": "Город",
      "code": "BR001"
    }
  ]
}
```

### Страницы

#### `/admin/managers`
Страница управления менеджерами с таблицей и модальными окнами для создания/редактирования.

**Компоненты:**
- `app/admin/managers/page.tsx` - серверный компонент с проверкой авторизации
- `app/admin/managers/ManagersClient.tsx` - клиентский компонент с UI

**Функции:**
- Просмотр списка менеджеров
- Поиск по имени, email, телефону
- Добавление нового менеджера
- Редактирование менеджера
- Удаление менеджера
- Назначение филиала

### База данных

Используются существующие модели из `prisma/schema.prisma`:

#### User (менеджеры)
```prisma
model User {
  id            String      @id @default(uuid())
  fullName      String
  email         String      @unique
  emailVerified Boolean     @default(false)
  role          UserRole    @default(user) // 'manager' для менеджеров
  phone         String?     @unique
  passwordHash  String
  status        UserStatus  @default(active)
  branchUsers   BranchUser[]
  // ...
}
```

#### BranchUser (связь менеджер-филиал)
```prisma
model BranchUser {
  id        String   @id @default(uuid())
  branchId  String
  userId    String
  branch    Branch   @relation(...)
  user      User     @relation(...)
  
  @@unique([branchId, userId])
}
```

## UI/UX

### Дизайн
Страница выполнена в едином стиле с остальной админ-панелью:
- Темная тема (#151b26, #252d3d)
- Фиолетовые акценты (#8b5cf6)
- Карточный дизайн с закругленными углами
- Hover-эффекты и плавные переходы

### Навигация
Добавлен пункт "Менеджеры" в боковую панель админ-панели между "Филиалы" и "Отчеты".

### Таблица менеджеров
Колонки:
- Менеджер (аватар + имя + ID)
- Email
- Телефон
- Филиал (с иконкой)
- Статус (цветной бадж)
- Действия (редактировать, удалить)

### Модальные окна

#### Добавление менеджера
Поля:
- Полное имя (обязательно)
- Email (обязательно)
- Телефон (необязательно)
- Пароль (обязательно, с показом/скрытием)
- Филиал (необязательно, выпадающий список)

#### Редактирование менеджера
Поля:
- Полное имя
- Email
- Телефон
- Новый пароль (необязательно)
- Статус (active/inactive/blocked)
- Филиал

## Безопасность

1. **Авторизация**: Все API endpoints проверяют наличие `admin_session` cookie
2. **Валидация**: Проверка обязательных полей и уникальности email/телефона
3. **Хеширование паролей**: Используется `hashPassword` из `lib/password.ts`
4. **Проверка роли**: Только пользователи с ролью `admin` могут управлять менеджерами

## Использование

### Добавление менеджера
1. Перейти в `/admin/managers`
2. Нажать "Добавить менеджера"
3. Заполнить форму
4. Нажать "Создать"

### Редактирование менеджера
1. Нажать иконку редактирования в строке менеджера
2. Изменить нужные поля
3. Нажать "Сохранить"

### Удаление менеджера
1. Нажать иконку удаления в строке менеджера
2. Подтвердить удаление

### Поиск менеджера
Ввести имя, email или телефон в поле поиска - таблица обновится автоматически.
