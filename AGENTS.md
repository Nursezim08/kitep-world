### Исправление: Отображение филиалов в корзине (15.05.2026)

**Проблема:**
- При оформлении заказа в корзине не отображались филиалы для выбора
- API endpoint `/api/user/branches` возвращал ошибку 500

**Причина:**
- Неправильное форматирование данных в API
- `workDays` пытались преобразовать из строки в массив через `split(',')`, но в БД это уже массив
- Неправильное форматирование времени через `new Date().toLocaleTimeString()`

**Решение:**
- ✅ Исправлено форматирование `workDays`: используется напрямую как массив
- ✅ Исправлено форматирование времени: `branch.openTime.toString().substring(0, 5)`
- ✅ Время корректно преобразуется из формата Time в строку "HH:MM"

**Изменения в API:**
```typescript
// Было (неправильно):
workDays: branch.workDays ? branch.workDays.split(',') : [],
openTime: branch.openTime
  ? new Date(branch.openTime).toLocaleTimeString('ru-RU', {...})
  : '09:00',

// Стало (правильно):
workDays: branch.workDays || [],
openTime: branch.openTime
  ? branch.openTime.toString().substring(0, 5)
  : '09:00',
```

**Результат:**
- ✅ API `/api/user/branches` возвращает 200 OK
- ✅ Филиалы корректно загружаются в корзине
- ✅ Компонент BranchSelect отображает список филиалов
- ✅ Пользователь может выбрать филиал для самовывоза

**Файлы:**
- `app/api/user/branches/route.ts` - Исправлено форматирование данных

### Анимация появления столбцов диаграммы при открытии фильтров (13.05.2026)

**Добавлена красивая анимация появления столбцов диаграммы:**

**Функционал:**
- Анимация срабатывает **только при первом открытии** модального окна фильтров
- Столбцы диаграммы появляются **по очереди** (слева направо)
- После завершения анимации она автоматически отключается
- При движении range slider анимация **не повторяется**

**Настройки анимации:**
```typescript
animations: {
  enabled: isFirstRender,           // Только при первом рендере
  easing: 'easeinout',              // Плавное ускорение/замедление
  speed: 800,                       // Длительность 800ms
  animateGradually: {
    enabled: true,                  // Постепенное появление
    delay: 30                       // Задержка между столбцами 30ms
  },
  dynamicAnimation: {
    enabled: false                  // Отключена при обновлениях
  }
}
```

**Логика работы:**
1. При монтировании компонента `isFirstRender = true`
2. Графики создаются с включенной анимацией
3. Столбцы появляются по очереди с задержкой 30ms
4. Через 1 секунду `isFirstRender` устанавливается в `false`
5. При следующем обновлении графиков анимация отключена

  - Рассчитывает средний рейтинг на сервере

  ```typescript
  interface Product {
    averageRating: number; // Средний рейтинг (0-5)

    }
  }
  ```

### Sticky позиционирование боковой панели (08.05.2026)

**Реализовано правильное поведение скролла:**

**Требование:**
- Боковая панель должна скроллиться вместе со страницей
- Когда контент боковой панели заканчивается, она должна остановиться (sticky)
- Основной контент должен продолжать скроллиться
- Не должно быть отступа под кнопкой "Выйти"

**Решение:**
- Добавлено `sticky top-[73px]` для прилипания к позиции 73px от верха
- Добавлено `self-start` для выравнивания по началу flex-контейнера
- Боковая панель останавливается когда достигает своего конца
- Основной контент продолжает скроллиться независимо

**Структура:**
```tsx
<div className="flex pt-[73px]">
  <aside className="px-4 pt-4 sticky top-[73px] self-start">
    {/* Навигация */}
    {/* Кнопка выхода */}
  </aside>
  <div className="flex-1">
    {/* Основной контент - продолжает скроллиться */}
  </div>
</div>
```

**Как работает:**
1. При скролле вниз боковая панель скроллится вместе со страницей
2. Когда низ боковой панели достигает viewport, она "прилипает" (`sticky`)
3. Основной контент продолжает скроллиться дальше
4. Нет лишнего пространства под кнопкой "Выйти"

### Исправление автоматического открытия выпадающего списка районов (06.05.2026)

- Это происходило из-за `useEffect`, который загружал районы при изменении города
- Пользователь видел открытый список, хотя не кликал на поле

**Решение:**
- Добавлен параметр `shouldOpen` в функцию `searchDistricts`
- При загрузке районов через `useEffect` передается `shouldOpen = false`
- Список открывается только при явном взаимодействии пользователя (фокус, ввод)

**Реализация:**

```typescript
// Поиск районов с параметром shouldOpen
const searchDistricts = async (query: string, shouldOpen: boolean = true) => {
  // ... код загрузки
  if (response.ok && data.districts) {
    setDistricts(data.districts);
    // Открываем список только если shouldOpen = true
    if (shouldOpen) {
      setIsOpen(data.districts.length > 0);
    }
  }
};

// Загрузка районов при изменении города (без автоматического открытия)
useEffect(() => {
  if (city) {
    searchDistricts('', false); // shouldOpen = false
  }
}, [city]);
```

- ✅ При изменении города районы загружаются в фоне
- ✅ Список НЕ открывается автоматически

### Автоматический скролл вверх при открытии модальных окон филиалов (06.05.2026)

**Исправлена проблема с отсутствием скролла при открытии модальных окон:**

**Проблема:**
- При открытии модального окна редактирования филиала верхняя часть (название и код) не была видна
- Модальное окно имеет внутренний скролл, а не скролл всей страницы
- Пользователь видел середину формы вместо начала

**Решение:**
- Добавлен `useRef` для контейнера модального окна
- При открытии модального окна скроллится его контейнер, а не вся страница
- Используется `scrollTop = 0` для мгновенного скролла к началу

**Реализация:**

*Для модального окна редактирования (встроенное):*
```typescript
const editModalRef = useRef<HTMLDivElement>(null);

const openEditModal = (branch: Branch) => {
  // ... установка данных
  setShowEditModal(true);
  
  setTimeout(() => {
    if (editModalRef.current) {
      editModalRef.current.scrollTop = 0;
    }
  }, 100);
};

// В JSX:
<div ref={editModalRef} className="fixed inset-0 ... overflow-y-auto">
```

*Для модального окна добавления (компонент):*
```typescript
const modalRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  if (isOpen && modalRef.current) {
    modalRef.current.scrollTop = 0;
  }
}, [isOpen]);

// В JSX:
<div ref={modalRef} className="fixed inset-0 ... overflow-y-auto">
```

**Файлы:**
- `app/admin/branches/BranchesClient.tsx` - Скролл для модального окна редактирования
- `app/admin/branches/AddBranchModal.tsx` - Скролл для модального окна добавления
