## Форматирование полей ввода

### Телефонные номера

Все поля ввода телефонных номеров в приложении используют компонент `PhoneInput` с автоматическим форматированием:

- **Автоматический префикс**: При фокусе на пустое поле автоматически добавляется `+996`
- **Формат**: `+996 XXX XXX XXX` (киргизский формат)
- **Валидация**: Проверка на 9 цифр после кода страны
- **Защита префикса**: Невозможно удалить `+996` с помощью Backspace или Delete
- **Визуальная обратная связь**: 
  - Красная подсветка при неполном номере
  - Зеленая галочка при корректном вводе
  - Подсказки о формате

**Использование:**
```tsx
import PhoneInput from '@/app/components/PhoneInput';

<PhoneInput
  label="Телефон"
  value={phone}
  onChange={(value) => setPhone(value)}
  placeholder="+996 XXX XXX XXX"
  required
/>
```

### Email адреса

Все поля ввода email используют компонент `EmailInput` с автоматическим форматированием:

- **Автоматическое форматирование**: Приведение к нижнему регистру
- **Удаление пробелов**: Автоматическое удаление всех пробелов
- **Валидация**: Проверка формата email (example@domain.com)
- **Визуальная обратная связь**:
  - Зеленая галочка при корректном email
  - Красный крестик при некорректном формате
  - Подсказки о формате

**Использование:**

<EmailInput

## Система входа для менеджеров

### Обзор

Менеджеры филиалов имеют доступ к специальной панели управления своим филиалом через двухфакторную аутентификацию с подтверждением по email.

### Процесс входа

1. **Ввод учетных данных** (`/manager/login`)
   - Email менеджера
   - Пароль (устанавливается администратором)

2. **Проверка прав доступа**
   - Проверка роли `manager`
   - Проверка привязки к филиалу
   - Проверка статуса аккаунта (active/blocked/inactive)

3. **Отправка кода подтверждения**
   - Генерация 6-значного кода
   - Отправка на email менеджера
   - Срок действия: 10 минут

4. **Подтверждение кода** (`/manager/verify`)
   - Ввод 6-значного кода из email
   - Возможность повторной отправки кода
   - Автоматический переход на страницу филиала

5. **Доступ к панели филиала** (`/manager/branch/[id]`)
   - Просмотр информации о филиале
   - Список менеджеров филиала
   - Управление операциями филиала

### API Endpoints

#### POST `/api/manager/login`
Вход менеджера с отправкой кода на email.

**Request:**
```json
{
  "email": "manager@example.com",
  "password": "password123"
}
```

**Response (Success):**
```json
{
  "success": true,
  "userId": "uuid",
  "branchId": "uuid",
  "message": "Код отправлен на email"
}
```

**Errors:**
- `invalidCredentials` - Неверный email или пароль
- `notManager` - Пользователь не является менеджером
- `noBranchAssigned` - Менеджер не привязан к филиалу
- `accountBlocked` - Аккаунт заблокирован
- `accountInactive` - Аккаунт неактивен

#### POST `/api/manager/verify-code`
Проверка кода подтверждения и создание сессии.

**Request:**
```json
{
  "userId": "uuid",
  "code": "123456"
}
```

**Response (Success):**
```json
{
  "success": true,
  "branchId": "uuid",
  "user": {
    "id": "uuid",
    "fullName": "Иван Иванов",
    "email": "manager@example.com",
    "role": "manager"
  }
}
```

**Errors:**
- `invalidCode` - Неверный код подтверждения
- `codeExpired` - Код подтверждения истек
- `notManager` - Пользователь не является менеджером
- `noBranchAssigned` - Менеджер не привязан к филиалу

#### POST `/api/manager/resend-code`
Повторная отправка кода подтверждения.

**Request:**
```json
{
  "userId": "uuid"
}
```

**Response (Success):**
```json
{
  "success": true,
  "message": "Код отправлен повторно"
}
```

### Безопасность

- **Двухфакторная аутентификация**: Пароль + код из email
- **Ограничение по времени**: Код действителен 10 минут
- **Проверка привязки**: Менеджер должен быть привязан к филиалу
- **JWT токены**: Безопасное хранение сессии в httpOnly cookies
- **Логирование**: Все попытки входа логируются

### Email шаблоны

Для входа менеджеров используется специальный email шаблон с синим дизайном (в отличие от фиолетового для обычных пользователей):

- Заголовок "Панель менеджера"
- Синяя цветовая схема (#3b82f6)
- 6-значный код с крупным шрифтом
- Информация о сроке действия (10 минут)
- Предупреждение о безопасности

### Страницы

#### `/manager/login`
- Форма входа с email и паролем
- Синий дизайн (отличается от админской панели)
- Валидация полей
- Обработка ошибок

#### `/manager/verify`
- Ввод 6-значного кода
- Кнопка повторной отправки
- Таймер действия кода
- Автоматический переход после успешной проверки

#### `/manager/branch/[id]`
- Информация о филиале (адрес, контакты, режим работы)
- Список менеджеров филиала
- Кнопка выхода
- Секция для управления операциями (расширяется в будущем)

### Интеграция с существующей системой

- Использует существующую таблицу `email_verifications` для хранения кодов
- Совместима с системой управления филиалами в админ-панели
- Использует общие утилиты аутентификации (`lib/auth.ts`)
- Интегрирована с системой отправки email (`lib/email.ts`)

### Разработка и тестирование

В режиме разработки (без настроенного SMTP):
- Коды выводятся в консоль сервера
- Все функции работают полностью
- Можно тестировать без реальной отправки email

Для production:
- Настроить SMTP в `.env` (SMTP_USER, SMTP_PASS)
- Коды отправляются на реальные email адреса

### Пример использования

1. Администратор создает менеджера в админ-панели и назначает его на филиал
2. Менеджер получает email и пароль от администратора
3. Менеджер переходит на `/manager/login`
4. Вводит email и пароль
5. Получает код на email
6. Вводит код на `/manager/verify`
7. Автоматически перенаправляется на страницу своего филиала
8. Может просматривать информацию и управлять операциями филиала


## Система управления категориями

### Обзор

Админ-панель включает полноценную систему управления категориями товаров с поддержкой иерархии и расширенной фильтрации.

### Фильтрация категорий

В разделе категорий (`/admin/categories`) доступна расширенная фильтрация по нескольким параметрам:

#### Параметры фильтрации

1. **Уровень вложенности:**
   - Все уровни (стандартный режим с навигацией)
   - Категории (уровень 0)
   - Подкатегории 1-5 уровня

2. **Статус:**
   - Все статусы
   - Активные
   - Неактивные

3. **Тип категории:**
   - Все типы
   - Родительские (с подкатегориями)
   - Конечные (без подкатегорий)

#### Интерфейс фильтрации

**Кнопка "Фильтры":**
- Расположена между поиском и кнопкой добавления
- Показывает счетчик активных фильтров в виде бейджа
- Фиолетовая подсветка при активных фильтрах
- Анимация иконки при открытии/закрытии

**Панель фильтров:**
- Раскрывается под строкой поиска
- Три выпадающих списка в адаптивной сетке
- Кнопка "Сбросить все" при наличии активных фильтров
- Список активных фильтров с возможностью удаления каждого

**Информационный блок:**
- Показывается при активных фильтрах
- Фиолетовый градиент
- Список активных фильтров в виде бейджей
- Кнопка "Сбросить" для сброса всех фильтров

**Карточки категорий:**
- Бейдж статуса (зеленый/серый)
- Бейдж родительской категории (фиолетовый)
- При активных фильтрах отключается навигация по клику
- Адаптивное отображение кнопок действий

### Особенности фильтрации

**При активных фильтрах:**
- Отображаются только категории, соответствующие всем критериям
- Скрываются breadcrumbs (хлебные крошки)
- Отключается навигация по клику на карточки
- Показывается информационный блок с активными фильтрами
- Счетчик активных фильтров на кнопке "Фильтры"
- Возможность сбросить отдельные фильтры или все сразу

**В стандартном режиме (без фильтров):**
- Навигация по уровням через клик на карточки
- Breadcrumbs для быстрого возврата
- Кнопка "Открыть" для категорий с подкатегориями
- Информационный блок при просмотре подкатегорий

### API Endpoint

#### GET `/api/admin/categories?level={number}`

Получение категорий с фильтрацией по уровню вложенности.

**Параметры:**
- `level` (optional) - Уровень вложенности (0 = категории, 1+ = подкатегории)
- `search` (optional) - Поиск по названию
- `parentId` (optional) - Фильтр по родительской категории (игнорируется при указании level)

**Примеры:**
```
GET /api/admin/categories?level=0          # Только категории
GET /api/admin/categories?level=1          # Подкатегории 1 уровня
GET /api/admin/categories?level=2&search=канцелярия  # Подкатегории 2 уровня с поиском
```

**Response:**
```json
[
  {
    "id": "uuid",
    "parentId": "uuid" | null,
    "image": "https://...",
    "status": "active",
    "level": 1,
    "translations": [
      { "locale": "ru", "name": "Ручки" },
      { "locale": "kg", "name": "Калемдер" }
    ],
    "_count": {
      "children": 5,
      "products": 12
    }
  }
]
```

### Логика вычисления уровня

Уровень категории вычисляется рекурсивно:
- Категория без родителя (parentId = null) → уровень 0
- Категория с родителем → уровень родителя + 1

**Пример иерархии:**
```
Канцелярия (уровень 0)
├── Ручки (уровень 1)
│   ├── Шариковые (уровень 2)
│   └── Гелевые (уровень 2)
└── Карандаши (уровень 1)
    └── Цветные (уровень 2)
```

## Админ-панель

### Разделы админ-панели

#### 1. Пользователи (`/admin/users`)

Управление пользователями системы:

- **Просмотр**: Таблица со всеми пользователями (имя, email, телефон, роль, статус, статистика)
- **Фильтрация**: По роли (admin, manager, user) и статусу (active, inactive, blocked)
- **Поиск**: По имени или email
- **Добавление**: Создание нового пользователя с указанием роли и статуса
- **Редактирование**: Изменение данных пользователя, роли, статуса, пароля
- **Удаление**: Блокировка пользователя (мягкое удаление)

**API Endpoints:**
- `GET /api/admin/users` - Список пользователей
- `POST /api/admin/users` - Создание пользователя
- `GET /api/admin/users/[id]` - Получение пользователя
- `PATCH /api/admin/users/[id]` - Обновление пользователя
- `DELETE /api/admin/users/[id]` - Блокировка пользователя

#### 2. Настройки (`/admin/settings`)

Управление настройками системы:

**Категории настроек:**
- **Общие**: Название сайта, описание, ключевые слова
- **Контакты**: Email, телефон, адрес
- **Социальные сети**: Facebook, Instagram, Telegram, WhatsApp
- **Рабочее время**: График работы
- **Оплата**: Finik Pay (Merchant ID, API Key)
- **AI Помощник**: Включение/выключение, модель (GPT-4, GPT-3.5), OpenAI API Key

**API Endpoints:**
- `GET /api/admin/settings` - Получение всех настроек
- `POST /api/admin/settings` - Обновление настроек

**Хранение:**
- Настройки хранятся в таблице `settings` в формате key-value
- Поддержка upsert (создание или обновление)

#### 3. Категории (`/admin/categories`)

Управление категориями товаров:

- Иерархическая структура (категории и подкатегории)
- Загрузка изображений в S3
- Мультиязычность (русский, кыргызский)
- Drag & drop для изображений
- Валидация квадратных PNG изображений

#### 4. Товары (`/admin/products`)

Управление товарами:

- Карточки товаров с изображениями
- Множественные изображения (галерея)
- Привязка к категориям
- Характеристики товаров
- Статусы (активен/неактивен)

#### 5. Филиалы (`/admin/branches`)

Управление филиалами:

- Информация о филиалах (адрес, контакты, режим работы)
- Назначение менеджеров
- Индивидуальное расписание для каждого дня недели
- Автозаполнение адресов (города, районы)

#### 6. Менеджеры (`/admin/managers`)

Управление менеджерами филиалов:

- Просмотр всех менеджеров
- Информация о привязанных филиалах
- Детальная информация о каждом менеджере

### Безопасность

- Проверка роли `admin` для доступа ко всем разделам
- Хеширование паролей через bcrypt
- JWT токены в httpOnly cookies
- Мягкое удаление (блокировка вместо физического удаления)
- Защита от удаления самого себя

### Дизайн

- Темная тема (фиолетовая цветовая схема)
- Адаптивный дизайн
- Модальные окна с backdrop blur
- Hover эффекты и анимации
- Иконки Lucide React
