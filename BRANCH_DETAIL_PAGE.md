# Страница деталей филиала

## ✅ Реализовано

Создана полноценная страница деталей филиала по адресу `/admin/branches/[id]`, которая отображает всю информацию о филиале.

## 📁 Структура файлов

- `app/admin/branches/[id]/page.tsx` - Серверный компонент
- `app/admin/branches/[id]/BranchDetailClient.tsx` - Клиентский компонент

## 🎯 Отображаемая информация

### 1. Основная информация о филиале

**Карточка с аватаром:**
- Аватар с первой буквой названия
- Название филиала
- Статус (Активен/Неактивен/Закрыт) с цветовой индикацией

**Детали:**
- 📋 **Код филиала** - уникальный идентификатор
- 📍 **Город** - местоположение
- 📧 **Email** - контактный email
- 📞 **Телефон** - контактный телефон
- 📅 **Дата создания** - когда был создан филиал
- 🕐 **Время работы** - часы открытия и закрытия (если указано)

### 2. Статистика (4 карточки)

1. **Менеджеры** 👥
   - Количество назначенных менеджеров
   - Синяя иконка

2. **Заказы** 🛒
   - Общее количество заказов
   - Зеленая иконка

3. **Выручка** 💰
   - Общая сумма всех заказов в тенге
   - Фиолетовая иконка

4. **Товары** 📦
   - Количество товаров в инвентаре
   - Оранжевая иконка

### 3. Адрес филиала

**Карточка с подробным адресом:**
- 🏢 **Район** - административный район
- 📍 **Полный адрес** - улица, дом
- 🗺️ **Координаты** - широта и долгота (если указаны)

### 4. Менеджеры филиала

**Список всех менеджеров:**
- Аватар с первой буквой имени
- Полное имя
- Email
- Статус (Активен/Неактивен/Заблокирован)
- Пустое состояние если нет менеджеров

### 5. Последние заказы

**Список последних 10 заказов:**
- Номер заказа
- Статус заказа (Оплачен/Завершен/Отменен)
- Дата и время создания
- Сумма заказа в тенге
- Пустое состояние если нет заказов

## 🎨 Дизайн

### Цветовая схема

**Статусы филиала:**
- 🟢 **Активен** - зеленый (`emerald-500`)
- ⚪ **Неактивен** - серый (`gray-500`)
- 🔴 **Закрыт** - красный (`red-500`)

**Статусы менеджеров:**
- 🟢 **Активен** - зеленый
- ⚪ **Неактивен** - серый
- 🔴 **Заблокирован** - красный

**Статусы заказов:**
- 🔵 **Оплачен** - синий (`blue-500`)
- 🟢 **Завершен** - зеленый (`emerald-500`)
- 🔴 **Отменен** - красный (`red-500`)

### Компоненты

- **Аватары** - градиентные круги с первой буквой
- **Карточки** - скругленные с границами
- **Иконки** - из lucide-react
- **Статусные бейджи** - с иконками и цветовой кодировкой
- **Сетка** - адаптивная (1-2-4 колонки)

### Адаптивность

- **Мобильные** (< 768px): 1 колонка
- **Планшеты** (768px - 1024px): 2 колонки
- **Десктоп** (> 1024px): 4 колонки для статистики, 2 для контента

## 🔐 Безопасность

### Авторизация
- Проверка JWT токена
- Проверка роли администратора
- Редирект если не авторизован

### Валидация
- Проверка существования филиала

## 📊 База данных

### Запрос данных

```typescript
const branchData = await prisma.branch.findUnique({
  where: { id },
  include: {
    branchUsers: {
      include: {
        user: {
          select: {
            id, fullName, email, phone, status
          }
        }
      }
    },
    orders: {
      select: {
        id, orderNumber, total, orderStatus, 
        paymentStatus, createdAt
      },
      orderBy: { createdAt: 'desc' },
      take: 10
    },
    _count: {
      select: {
        branchUsers, orders, inventory
      }
    }
  }
});
```

## 🔄 Обновление данных

Данные загружаются при каждом посещении страницы (серверный рендеринг).

Для обновления данных:
- Обновить страницу (F5)
- Перейти на другую страницу и вернуться

## 📝 Примеры использования

## 📊 Метрики

На странице отображаются следующие метрики:
- **Количество менеджеров** - из `_count.branchUsers`
- **Количество заказов** - из `_count.orders`
- **Общая выручка** - сумма всех `order.total`
- **Количество товаров** - из `_count.inventory`

Все метрики вычисляются в реальном времени из базы данных.
