### 2. Автоматическое удаление старых изображений

#### При изменении изображения:
```typescript
// 1. Загружаем новое изображение в S3
const newImageUrl = await uploadImageToS3(image, 'categories');

// 2. Удаляем старое изображение из S3
if (existingCategory.image && isS3Image(existingCategory.image)) {
  await deleteImageFromS3(existingCategory.image);
}

// 3. Обновляем URL в БД
updateData.image = newImageUrl;
```

**Результат:**
- ✅ Старое изображение удалено из S3
- ✅ Новое изображение загружено в S3
- ✅ Хранилище не засоряется

#### При удалении категории:
```typescript
// 1. Получаем категорию
const category = await prisma.category.findUnique({
  where: { id },
  select: { image: true },
});

// 2. Мягкое удаление категории
await prisma.category.update({
  where: { id },
  data: { status: 'deleted' },
});

// 3. Удаляем изображение из S3
if (category?.image && isS3Image(category.image)) {
  await deleteImageFromS3(category.image);
}
```

**Результат:**
- ✅ Категория помечена как удалённая
- ✅ Изображение удалено из S3
- ✅ Хранилище очищено

## Процесс работы

### Создание категории
```
Пользователь загружает PNG
    ↓
Конвертация в base64
    ↓
POST /api/admin/categories
    ↓
uploadImageToS3(base64, 'categories')
    ↓
Сохранение URL в БД
    ↓
Возврат категории с URL
```

### Изменение изображения
```
Пользователь загружает новое PNG
    ↓
Конвертация в base64
    ↓
PATCH /api/admin/categories/[id]
    ↓
uploadImageToS3(newImage, 'categories')  ← Загрузка нового
    ↓
deleteImageFromS3(oldImageUrl)           ← Удаление старого
    ↓
Обновление URL в БД
    ↓
Возврат обновлённой категории
```

### Удаление категории
```
Пользователь удаляет категорию
    ↓
DELETE /api/admin/categories/[id]
    ↓
Проверка: нет подкатегорий и товаров
    ↓
Мягкое удаление: status = 'deleted'
    ↓
deleteImageFromS3(imageUrl)              ← Удаление из S3
    ↓
Возврат успеха
```
