# Инструкция по тестированию системы кодов заказов

## Подготовка

1. Убедитесь, что сервер запущен: `npm run dev`
2. Откройте два окна браузера:
   - Окно 1: Панель менеджера
   - Окно 2: Приложение пользователя

## Шаг 1: Создание заказа (пользователь)

1. Войдите как пользователь: `http://localhost:3000/login`
2. Перейдите в каталог: `http://localhost:3000/catalog`
3. Добавьте товары в корзину
4. Перейдите в корзину: `http://localhost:3000/cart`
5. Нажмите "Оформить заказ"
6. Выберите филиал для самовывоза
7. Нажмите "Перейти к оплате"
8. Оплатите заказ через Finik Pay (тестовый режим)
9. После успешной оплаты заказ получит статус `paid`

## Шаг 2: Просмотр кода заказа (пользователь)

1. Перейдите на страницу заказов: `http://localhost:3000/orders`
2. Найдите свой заказ в списке
3. **Кликните по карточке заказа**
4. Откроется модальное окно с деталями
5. **Запомните 5-значный код** в фиолетовом блоке

**Пример:**
```
┌─────────────────────────────────────┐
│ Код для получения заказа            │
│                                     │
│         85808                       │
│                                     │
│ Назовите этот код менеджеру        │
└─────────────────────────────────────┘
```

## Шаг 3: Выдача заказа (менеджер)

1. Войдите как менеджер: `http://localhost:3000/manager/login`
2. Введите email и пароль менеджера
3. Введите код из email
4. Перейдите на страницу заказов: `http://localhost:3000/manager/orders`
5. Найдите заказ со статусом "Оплачен" (синий бейдж)
6. Нажмите кнопку **"Выдать"**
7. Откроется модальное окно с деталями заказа

**Модальное окно содержит:**
- ✅ Информацию о клиенте
- ✅ Список всех товаров с изображениями
- ✅ Общую сумму заказа
- ✅ Поле ввода кода подтверждения

## Шаг 4: Ввод кода и выдача

1. В поле "Код заказа" введите **5 цифр**, которые видел пользователь
2. Кнопка "Выдать заказ" станет активной
3. Нажмите "Выдать заказ"
4. Заказ переведется в статус `completed` (зеленый бейдж)

## Тестовые сценарии

### ✅ Успешная выдача

1. Пользователь показывает код: `85808`
2. Менеджер вводит: `85808`
3. Результат: Заказ выдан, статус `completed`

### ❌ Неправильный код

1. Пользователь показывает код: `85808`
2. Менеджер вводит: `12345`
3. Результат: Ошибка "Неверный код заказа"

### ⚠️ Неполный код

1. Менеджер вводит: `858` (только 3 цифры)
2. Результат: Кнопка "Выдать заказ" неактивна

## Проверка результата

### Пользователь

1. Обновите страницу заказов: `http://localhost:3000/orders`
2. Статус заказа изменился на "Завершен" (зеленый)
3. Кликните по заказу - код все еще виден

### Менеджер

1. Обновите страницу заказов: `http://localhost:3000/manager/orders`
2. Заказ исчез из списка "Оплачен"
3. Заказ появился в статистике "Завершено"

## Извлечение кода из номера заказа

**Формула:**
```
Номер заказа: ORD-1779710728185-808
↓ Убираем дефисы
ORD1779710728185808
↓ Убираем буквы
1779710728185808
↓ Берем последние 5 цифр
85808
```

**Примеры:**
- `ORD-1779710728185-808` → `85808`
- `ORD-1779710728185-123` → `85123`
- `ORD-1779710728185-999` → `85999`

## Возможные проблемы

### Проблема: Кнопка "Выдать" не появляется

**Решение:** Убедитесь, что заказ имеет статус `paid` (синий бейдж)

### Проблема: Модальное окно не открывается (пользователь)

**Решение:** Убедитесь, что вы кликаете по карточке заказа, а не по кнопкам внутри

### Проблема: Ошибка "Неверный код заказа"

**Решение:** 
1. Проверьте, что вводите именно последние 5 цифр
2. Убедитесь, что номер заказа совпадает
3. Проверьте консоль браузера на ошибки

### Проблема: Детали заказа не загружаются

**Решение:**
1. Проверьте консоль браузера (F12)
2. Убедитесь, что API endpoint работает: `/api/manager/orders/[id]`
3. Проверьте, что менеджер привязан к филиалу заказа
