# Быстрый старт: Управление менеджерами

## Что добавлено

✅ Раздел "Менеджеры" в боковой панели админ-панели  
✅ Страница со списком всех менеджеров  
✅ Возможность добавления нового менеджера  
✅ Редактирование данных менеджера  
✅ Удаление менеджера  
✅ Назначение менеджера на филиал  
✅ Поиск по имени, email, телефону  
✅ Локализация (русский и кыргызский)

## Как использовать

### 1. Войти в админ-панель
```
http://localhost:3000/admin/login
```

### 2. Перейти в раздел "Менеджеры"
В боковой панели нажать на пункт "Менеджеры" (между "Филиалы" и "Отчеты")

### 3. Добавить менеджера
1. Нажать кнопку "Добавить менеджера"
2. Заполнить форму:
   - **Полное имя** (обязательно)
   - **Email** (обязательно, уникальный)
   - **Телефон** (необязательно)
   - **Пароль** (обязательно)
   - **Филиал** (необязательно)
3. Нажать "Создать"

### 4. Редактировать менеджера
1. Нажать иконку карандаша в строке менеджера
2. Изменить нужные поля
3. Нажать "Сохранить"

### 5. Удалить менеджера
1. Нажать иконку корзины в строке менеджера
2. Подтвердить удаление

## API Endpoints

### Получить всех менеджеров
```http
GET /api/admin/managers
```

### Создать менеджера
```http
POST /api/admin/managers
Content-Type: application/json

{
  "fullName": "Иван Иванов",
  "email": "manager@example.com",
  "phone": "+996555123456",
  "password": "password123",
  "branchId": "uuid-филиала" // необязательно
}
```

### Обновить менеджера
```http
PATCH /api/admin/managers/{id}
Content-Type: application/json

{
  "fullName": "Новое имя",
  "email": "newemail@example.com",
  "phone": "+996555654321",
  "password": "newpassword", // необязательно
  "status": "active", // active | inactive | blocked
  "branchId": "uuid-филиала" // необязательно
}
```

### Удалить менеджера
```http
DELETE /api/admin/managers/{id}
```

### Получить список филиалов
```http
GET /api/admin/branches
```

## Структура БД

### User (менеджер)
- `role: 'manager'` - роль менеджера
- `emailVerified: true` - email подтвержден автоматически
- `status: 'active'` - статус по умолчанию

### BranchUser (связь)
- Связывает менеджера с филиалом
- Один менеджер может быть назначен на один филиал
- При удалении менеджера связь удаляется автоматически

## Статусы менеджера

- **active** (Активен) - менеджер может работать
- **inactive** (Неактивен) - менеджер временно отключен
- **blocked** (Заблокирован) - менеджер заблокирован

## Особенности

1. **Email уникален** - нельзя создать двух менеджеров с одинаковым email
2. **Телефон уникален** - если указан, должен быть уникальным
3. **Пароль хешируется** - пароли хранятся в зашифрованном виде
4. **Автоматическая верификация** - менеджеры создаются с подтвержденным email
5. **Один филиал** - менеджер может быть назначен только на один филиал
6. **Каскадное удаление** - при удалении менеджера удаляются все его связи
