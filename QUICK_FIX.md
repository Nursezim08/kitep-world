## Если не работает

### Проверьте Network tab (F12 → Network)
1. Найдите запрос к `/api/manager/orders/[id]`
2. Проверьте статус ответа:
   - **200** - успешно
   - **401** - не авторизован
   - **404** - заказ не найден
   - **500** - ошибка сервера

### Проверьте Response (ответ сервера)
Кликните на запрос → вкладка "Response"

**Успешный ответ (200):**
```json
{
  "id": "...",
  "orderNumber": "ORD-...",
  "user": {...},
  "orderItems": [...]
}
```

**Ошибка (401):**
```json
{
  "error": "Unauthorized"
}
```

**Ошибка (404):**
```json
{
  "error": "Branch not found"
}
```
или
```json
{
  "error": "Order not found"
}
```

## Частые проблемы

### Проблема 1: "Branch not found"
**Причина:** Менеджер не привязан к филиалу
**Решение:** 
1. Проверьте таблицу `branch_users`
2. Убедитесь, что есть запись с `user_id` менеджера

### Проблема 2: "Order not found"
**Причина:** Заказ не принадлежит филиалу менеджера
**Решение:**
1. Проверьте `branch_id` заказа
2. Убедитесь, что он совпадает с филиалом менеджера

### Проблема 3: Пустое модальное окно
**Причина:** Ошибка при загрузке данных
**Решение:**
1. Проверьте консоль на наличие ошибок
2. Проверьте Network tab
3. Проверьте логи сервера

### Проблема 4: Изображения не загружаются
**Причина:** Неправильные URL изображений
**Решение:**
1. Проверьте таблицу `product_images`
2. Убедитесь, что `image_url` содержит валидные URL

## Дополнительная отладка

### Проверка в базе данных

**Проверка менеджера:**
```sql
SELECT * FROM users WHERE id = '{managerId}' AND role = 'manager';
```

**Проверка привязки к филиалу:**
```sql
SELECT * FROM branch_users WHERE user_id = '{managerId}';
```

**Проверка заказа:**
```sql
SELECT * FROM orders WHERE id = '{orderId}';
```

**Проверка товаров:**
```sql
SELECT oi.*, p.sku, pt.name 
FROM order_items oi
JOIN products p ON oi.product_id = p.id
LEFT JOIN product_translations pt ON p.id = pt.product_id AND pt.locale = 'ru'
WHERE oi.order_id = '{orderId}';
```
