# Защита Google аккаунтов от входа по паролю

## Обзор

Реализована защита, которая предотвращает вход по паролю для пользователей, зарегистрированных через Google OAuth. Это повышает безопасность и обеспечивает правильный способ аутентификации.

## Как это работает

### Сценарий 1: Попытка входа по паролю

1. Пользователь регистрируется через Google
2. В базе данных создается запись с `googleId`
3. Пользователь пытается войти через форму логина с паролем
4. Система проверяет наличие `googleId`
5. Если `googleId` существует → показывается модальное окно
6. Пользователь перенаправляется на Google OAuth

### Сценарий 2: Попытка регистрации с существующим Google email

1. Пользователь уже зарегистрирован через Google
2. Пользователь пытается зарегистрироваться с тем же email и паролем
3. Система находит существующего пользователя с `googleId`
4. Показывается модальное окно с предложением войти через Google

## Реализация

### Backend (API)

#### `/api/auth/login/route.ts`

```typescript
// Проверка: если пользователь зарегистрирован через Google
if (user.googleId) {
  return NextResponse.json(
    { 
      error: 'Этот аккаунт зарегистрирован через Google',
      useGoogleAuth: true 
    },
    { status: 400 }
  );
}
```

#### `/api/auth/register/route.ts`

```typescript
if (existingUser) {
  // Если пользователь зарегистрирован через Google
  if (existingUser.googleId) {
    return NextResponse.json(
      { 
        error: 'Пользователь с таким email уже существует',
        useGoogleAuth: true 
      },
      { status: 409 }
    );
  }
  
  return NextResponse.json(
    { error: 'Пользователь с таким email уже существует' },
    { status: 409 }
  );
}
```

## Поток пользователя

### Вход (Login)

```
1. Пользователь вводит email и пароль
   ↓
2. POST /api/auth/login
   ↓
3. Проверка user.googleId
   ↓
4. Если googleId существует:
   ├─→ Возврат { useGoogleAuth: true }
   ├─→ Показ модального окна
   └─→ Перенаправление на Google OAuth
   
5. Если googleId отсутствует:
   └─→ Обычный вход по паролю
```

### Регистрация (Register)

```
1. Пользователь заполняет форму регистрации
   ↓
2. POST /api/auth/register
   ↓
3. Проверка существующего пользователя
   ↓
4. Если пользователь существует с googleId:
   ├─→ Возврат { useGoogleAuth: true }
   ├─→ Показ модального окна
   └─→ Перенаправление на Google OAuth
   
5. Если пользователь не существует:
   └─→ Обычная регистрация
```

## Безопасность

### Преимущества

1. **Защита от компрометации пароля**
   - Google аккаунты не имеют пароля в системе
   - Невозможно войти с украденным паролем

2. **Единый способ аутентификации**
   - Пользователь всегда использует Google OAuth
   - Нет путаницы с разными способами входа

3. **Лучший UX**
   - Понятное сообщение о способе входа
   - Прямая ссылка на правильный метод

### FAQ

**Q: Почему я не могу войти с паролем?**
A: Ваш аккаунт зарегистрирован через Google. Используйте кнопку "Войти через Google" для входа.

**Q: Я забыл, что регистрировался через Google. Что делать?**
A: Просто попробуйте войти с паролем - система автоматически покажет, что нужно использовать Google.

**Q: Могу ли я добавить пароль к Google аккаунту?**
A: В текущей версии это не поддерживается. Используйте Google OAuth для входа.
