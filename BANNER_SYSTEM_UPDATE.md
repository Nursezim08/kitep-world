# Обновление системы баннеров

## Обзор изменений

Система баннеров обновлена для поддержки отдельных изображений для десктопа и мобильных устройств.

## Требования к изображениям

### Десктоп
- **Размер:** 1920x600 пикселей (строго)
- **Формат:** Любой (PNG, JPG, WEBP и т.д.)
- **Максимальный размер файла:** 10MB
- **Пропорции:** 16:9

### Мобильные устройства
- **Размер:** 768x1024 пикселей (строго)
- **Формат:** Любой (PNG, JPG, WEBP и т.д.)
- **Максимальный размер файла:** 10MB
- **Пропорции:** 3:4

## Хранение данных

Изображения хранятся в базе данных в формате **base64** (JSON):

```json
{
  "id": "uuid",
  "title": "Название баннера",
  "desktopImage": "data:image/png;base64,...",
  "mobileImage": "data:image/png;base64,...",
  "url": "https://example.com/promo",
  "status": "active"
}
```

## Логика отображения

Система автоматически определяет тип устройства и показывает соответствующее изображение:

```typescript
const isMobile = window.innerWidth < 1024; // lg breakpoint

// Отображение
<img src={isMobile ? banner.mobileImage : banner.desktopImage} />
```

### Breakpoints
- **Мобильные:** < 1024px (показывается mobileImage)
- **Десктоп:** ≥ 1024px (показывается desktopImage)

## Валидация

### При загрузке изображения

1. **Проверка типа файла:**
   ```typescript
   if (!file.type.startsWith('image/')) {
     error: 'Допускаются только изображения'
   }
   ```

2. **Проверка размера файла:**
   ```typescript
   if (file.size > 10 * 1024 * 1024) {
     error: 'Размер изображения не должен превышать 10MB'
   }
   ```

3. **Проверка размеров изображения:**
   ```typescript
   // Для десктопа
   if (img.width !== 1920 || img.height !== 600) {
     error: 'Изображение для десктопа должно быть 1920x600px'
   }
   
   // Для мобильных
   if (img.width !== 768 || img.height !== 1024) {
     error: 'Изображение для мобильных должно быть 768x1024px'
   }
   ```

### При сохранении

API проверяет наличие обоих изображений:

```typescript
if (!desktopImage) {
  return { error: 'Изображение для десктопа обязательно' }
}

if (!mobileImage) {
  return { error: 'Изображение для мобильных обязательно' }
}
```

## API Endpoints

### POST /api/admin/banners
Создание нового баннера

**Request:**
```json
{
  "title": "Летняя распродажа",
  "desktopImage": "data:image/png;base64,...",
  "mobileImage": "data:image/png;base64,...",
  "url": "https://example.com/promo",
  "status": "active"
}
```

**Response:**
```json
{
  "banner": {
    "id": "uuid",
    "title": "Летняя распродажа",
    "desktopImage": "data:image/png;base64,...",
    "mobileImage": "data:image/png;base64,...",
    "url": "https://example.com/promo",
    "status": "active"
  }
}
```

### PATCH /api/admin/banners/[id]
Обновление баннера

**Request:** (аналогично POST)

### GET /api/admin/banners
Получение всех баннеров

**Response:**
```json
{
  "banners": [
    {
      "id": "uuid",
      "title": "Летняя распродажа",
      "desktopImage": "data:image/png;base64,...",
      "mobileImage": "data:image/png;base64,...",
      "url": "https://example.com/promo",
      "status": "active"
    }
  ]
}
```

## Миграция существующих данных

Если у вас есть существующие баннеры с одним изображением, необходимо:

1. Создать два варианта изображения (1920x600 и 768x1024)
2. Обновить каждый баннер через админку
3. Загрузить оба изображения

## Преимущества новой системы

✅ **Оптимальное отображение** - каждое устройство получает изображение правильного размера
✅ **Лучшая производительность** - мобильные устройства не загружают большие десктопные изображения
✅ **Гибкость дизайна** - можно создавать разные композиции для разных устройств
✅ **Строгая валидация** - гарантирует правильные размеры изображений
✅ **Автоматическое переключение** - система сама определяет тип устройства

## Рекомендации по дизайну

### Десктоп (1920x600)
- Горизонтальная композиция
- Текст можно размещать слева или справа
- Больше пространства для деталей

### Мобильные (768x1024)
- Вертикальная композиция
- Текст лучше размещать в центре или сверху
- Крупные элементы для удобства чтения

## Troubleshooting

### Ошибка: "Изображение должно быть 1920x600px"
- Проверьте размеры изображения в графическом редакторе
- Используйте точные размеры, не приблизительные

### Ошибка: "Размер изображения не должен превышать 10MB"
- Оптимизируйте изображение (сжатие, формат)
- Используйте онлайн-сервисы для сжатия (TinyPNG, Squoosh)
- Рассмотрите использование формата WebP для лучшего сжатия

### Баннер не отображается на мобильных
- Убедитесь, что загружено изображение для мобильных
- Проверьте размер изображения (768x1024)
- Очистите кэш браузера

### Баннер не переключается при изменении размера окна
- Обновите страницу
- Проверьте консоль браузера на ошибки
