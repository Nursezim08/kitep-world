│                    СОЗДАНИЕ ЗАКАЗА                              │
└─────────────────────────────────────────────────────────────────┘

        ↓                                           ↓

│  КОД: 85808       │                    │  Введите код:        │
│                   │                    │  [_____]             │

## Алгоритм извлечения кода

```
Номер заказа: ORD-1779710728185-808
                    ↓
        Шаг 1: Убираем дефисы
                    ↓
        ORD1779710728185808
                    ↓
        Шаг 2: Убираем буквы
                    ↓
        1779710728185808
                    ↓
        Шаг 3: Берем последние 5 цифр
                    ↓
              85808
```

## Состояния заказа

```
┌──────────┐
│  pending │  Ожидает оплаты
└────┬─────┘
     ↓
┌────────┐
│  paid  │  Оплачен (синий) → Можно выдать
└────┬───┘
     ↓ (Менеджер вводит код)
     ↓
┌────────────┐
│ completed  │  Завершен (зеленый) → Выдан клиенту
└────────────┘

     ↓ (Альтернативный путь)
     ↓
┌────────────┐
│ cancelled  │  Отменен (красный)
└────────────┘
```

## Безопасность

```
┌─────────────────────────────────────────────────────────────────┐
│                    ПРОВЕРКИ БЕЗОПАСНОСТИ                        │
└─────────────────────────────────────────────────────────────────┘

1. Авторизация менеджера
   ├─ Проверка JWT токена
   ├─ Проверка роли (manager)
   └─ Проверка привязки к филиалу

2. Доступ к заказу
   ├─ Заказ принадлежит филиалу менеджера
   ├─ Статус заказа = paid
   └─ Заказ не удален

3. Валидация кода
   ├─ Длина = 5 символов
   ├─ Только цифры
   └─ Совпадает с последними 5 цифрами номера

4. Выдача заказа
   ├─ Код верный
   ├─ Статус = paid
   └─ Изменение статуса → completed
```

## Обработка ошибок

```
┌─────────────────────────────────────────────────────────────────┐
│                    ВОЗМОЖНЫЕ ОШИБКИ                             │
└─────────────────────────────────────────────────────────────────┘

1. "Неверный код заказа"
   ├─ Причина: Введенный код не совпадает
   └─ Действие: Попробовать снова

2. "Unauthorized"
   ├─ Причина: Менеджер не авторизован
   └─ Действие: Войти в систему

3. "Order not found"
   ├─ Причина: Заказ не найден или не принадлежит филиалу
   └─ Действие: Проверить номер заказа

4. "Не удалось загрузить детали заказа"
   ├─ Причина: Ошибка сети или сервера
   └─ Действие: Обновить страницу

5. "Ошибка сети"
   ├─ Причина: Нет подключения к интернету
   └─ Действие: Проверить соединение
```

## Производительность

```
┌─────────────────────────────────────────────────────────────────┐
│                    ОПТИМИЗАЦИЯ                                  │
└─────────────────────────────────────────────────────────────────┘

1. Загрузка деталей заказа
   ├─ Запрос: GET /api/manager/orders/[id]
   ├─ Время: ~200-500ms
   └─ Кеширование: Нет (всегда актуальные данные)

2. Выдача заказа
   ├─ Запрос: PATCH /api/manager/orders/[id]/complete
   ├─ Время: ~100-300ms
   └─ Обновление: Автоматическое обновление списка

3. Модальное окно
   ├─ Анимация: backdrop-blur-sm
   ├─ Прокрутка: overflow-y-auto
   └─ Размер: max-w-2xl, max-h-90vh
```
